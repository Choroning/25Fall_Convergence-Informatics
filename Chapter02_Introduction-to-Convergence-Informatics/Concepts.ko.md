## 1. 융합정보학이란

### 1.1 정보학의 정의

- **Data Science == Informatics =~ Machine Learning**
- 정보학(Informatics)은 방대한 양의 데이터로부터 (자동적 또는 계산적으로) 지식을 관리, 조작, 추출, 해석하는 분야이다
- 데이터 사이언스(Data Science)와 정보학(Informatics)은 데이터 기반 문제를 해결하는 것을 목표로 하는 **다학제적(multidisciplinary)** 학문 분야이다

이 용어는 독일어 *Informatik* (Steinbuch, K. 1957)에서 유래하며, 영어의 Computer Science, 프랑스어의 *Informatique* (Philippe, 1692), 그리고 영어의 *Informatics* (Walter, 1967)에 해당한다.

- 데이터 사이언스의 원칙은 **모든 데이터 — 크든 작든**에 적용된다

### 1.2 융합(다학제적) 정보학

융합정보학(Convergence Informatics)은 다양한 분야의 지식을 통합하여 데이터에서 의미 있는 통찰을 추출한다. 다음과 같은 분야를 기반으로 한다:

- 데이터 사이언스의 개념
- R 프로그래밍
- 정보학을 위한 통계적 기술
- 정보학을 위한 계산 기술
- 생물정보학(Bioinformatics) 및 전산 생물학(Computational Biology) (후반부 강의 초점 — 컴퓨터과학, 화학, 공학, 수학, 생화학, 통계학, 생물학의 다학제적 교차점)
- 정보학의 응용

### 1.3 데이터 사이언스의 핵심 요소

데이터 사이언스는 통계학, 패턴 인식, 머신러닝, AI, 데이터 마이닝, 데이터베이스, 시각화 등 여러 분야 간의 상호작용을 포함한다. 과학, 공학, 경제학, 정치학, 금융, 교육 분야의 이론과 기법이 모두 하나로 결합된다.

### 1.4 데이터 사이언티스트

- **Data Scientist == Informatician** — **"21세기의 가장 섹시한 직업"**으로 불린다
- 데이터 사이언티스트는 빅데이터가 제시하는 기회를 실현하는 핵심 인물이다. 이들은 데이터에 구조를 부여하고, 매력적인 패턴을 발견하며, 제품, 프로세스, 의사결정에 대해 조언한다.

**데이터 사이언티스트 / 정보학자에게 필요한 역량:**

| 분류 | 세부 내용 |
|------|-----------|
| 수학 (Mathematics) | 수학 및 응용수학 |
| 통계학 (Statistics) | 통계학 및 응용통계학 |
| 프로그래밍 (Programming) | C/C++, Java, R, Python, Julia, SQL, JavaScript |
| 데이터 마이닝 (Data Mining) | 패턴 발견 및 지식 추출 |
| 데이터베이스 (Database) | 데이터베이스 저장 및 관리 |
| 머신러닝 (Machine Learning) | 머신러닝 및 발견 |

> **핵심 포인트:** 융합정보학은 본질적으로 다학제적(multidisciplinary)이다 — 수학, 통계학, 탄탄한 프로그래밍, 데이터 마이닝, 데이터베이스 관리, 머신러닝 역량이 요구된다. 모든 학문 분야가 서로 상호작용한다.

## 2. 통계학의 기본 개념

### 2.1 모집단과 표본

| 용어 | 정의 |
|------|------|
| **모집단 (Population)** | 주어진 주제와 관련된 모든 가능한 사건/결과의 합집합 |
| **표본 (Sample)** | 분석에 사용되는 모집단의 부분집합 |

### 2.2 중심 경향성 측도

- **평균 (Average, Arithmetic Mean)**: 모든 값의 합을 값의 개수로 나눈 것
- **중앙값 (Median)**: 데이터를 순서대로 정렬했을 때 중간에 위치하는 값

### 2.3 산포도 측도

- **범위 (Range)**: 최댓값과 최솟값의 차이

- **모분산 (Population Variance)**:

  V = sigma^2 = Sum(xi - mu)^2 / N

- **표본분산 (Sample Variance)**:

  Vs = sigma_s^2 = Sum(xi - x_bar)^2 / (n - 1)

- **표준편차 (Standard Deviation)**: 분산의 제곱근

> **핵심 포인트:** 표본분산은 모집단 분산의 비편향 추정량(unbiased estimate)을 제공하기 위해 **N** 대신 **(n - 1)**로 나눈다 (베셀 보정, Bessel's correction).

### 2.4 표본추출 개념

| 개념 | 설명 |
|------|------|
| **대표성 (Representativeness)** | 표본이 모집단의 특성을 얼마나 잘 반영하는지의 정도 |
| **표본오차 (Sampling Error)** | 표본을 추출할 때 자연스럽게 발생하는 무작위 변동 |
| **표본편향 (Sampling Bias)** | 표본이 모집단을 잘못 대표하게 만드는 체계적 왜곡 |

> **핵심 포인트:** 표본오차(Sampling Error)는 피할 수 없는 무작위 변동이고, 표본편향(Sampling Bias)은 식별하고 교정해야 하는 데이터 수집 과정에서의 체계적 문제이다.

## 3. 데이터 유형과 구조

### 3.1 정형 데이터 vs 비정형 데이터

| 유형 | 설명 | 예시 |
|------|------|------|
| **정형 데이터 (Structured)** | 고정된 스키마로 행과 열에 구조화된 데이터 | 데이터베이스, 스프레드시트, CSV 파일 |
| **비정형 데이터 (Unstructured)** | 사전 정의된 데이터 모델이 없는 데이터 | 이메일, 이미지, 비디오, 웹 페이지, 센서 출력 |

**정형 데이터 요건:**
- 정의된 행과 열의 수로 관계를 표현하는 **스키마(schema)**가 있어야 한다
- 모든 값이 **실수(real number)**여야 한다
- **행렬(matrix)** 형태여야 한다
- 데이터에 행의 수, 열의 수, 스키마 + 행렬 형태 데이터 + 모든 실수 값이 있으면 정형 데이터이다

비정형 데이터는 전통적인 데이터베이스에 존재하지 않으며, 내부 구조가 있을 수 있으나 관계형 모델에 맞지 않고, 전 세계 데이터의 약 **85%**를 차지한다.

비정형 데이터의 체계적 활용은 **빅데이터(Big Data)**의 핵심 과제이다.

### 3.2 확률변수 (Random Variable, R.V.)

확률변수(Random Variable)는 실제 현상을 데이터 값으로 매핑하는 함수이다. 두 가지 주요 유형이 있다:

| 유형 | 다른 명칭 | 목적 | 예시 |
|------|-----------|------|------|
| **연속형 (Continuous)** (수치형) | 양적 (Quantitative) | 계산 (수량) | 키, 몸무게, 온도 |
| **범주형 (Categorical)** (특성형) | 질적 (Qualitative) | 분류 (등급) | 성별, 색상, 결혼 여부 |
| **이산형 (Discrete)** | (셀 수 있는 값의 부분집합) | 문맥에 따라 다름 | 정수 값 카운트, 만 나이 |

- **연속형 속성(Continuous attributes)**은 실수 값을 가지며, 셀 수 없이 무한히 많은 값을 취할 수 있다 (부동소수점 변수로 표현)
- **이산형 속성(Discrete attributes)**은 유한하거나 셀 수 있는 무한 집합의 값을 갖는다 (보통 정수로 표현); 이진 속성(binary attributes)은 특수한 형태이다
- **이산형 vs 범주형의 미묘한 차이**: 정수형 데이터를 어떻게 해석할지는 분석 목적에 따라 달라진다:
  - **연속형 해석** — 숫자를 계산을 위한 수량으로 취급 (예: 평균 계산이나 회귀 분석을 위한 22세)
  - **범주형 해석** — 숫자를 분류를 위한 등급으로 취급 (예: 그룹 빈도 집계나 그룹 간 특성 비교를 위한 "20대" 그룹)
- 연속형 변수는 **구간화(binning)**를 통해 이산형 변수로 변환할 수 있다
- 변수가 연속형인지 범주형인지는 **매핑 과정**에 의해 결정된다

### 3.3 자유도 문제 (Freedom Degree Issue)

데이터를 범주화하려면 의미 있는 범주를 설정하기 위해 **최소 3개의 동일 값 표본**이 필요하다. 이는 통계 분석의 자유도(degrees of freedom)와 관련된다.

**예시 — 서로 다른 해상도로 표현된 나이:**

| 대상 | Age_Cont | Age_Cate1 | Age_Cate2 |
|------|----------|-----------|-----------|
| Student_1 | 10.41412 | 10 | 10s |
| Student_2 | 20.5213 | 20 | 20s |
| Student_3 | 22.2139123 | 22 | 20s |
| Student_4 | 23.1213 | 23 | 20s |

| | Age_Cont | Age_Cate1 | Age_Cate2 |
|---|----------|-----------|-----------|
| **해상도** | 최고 | | 최저 |
| **유형** | 연속형 | 연속형 / 범주형 | 범주형 |

- (10, 10, 10, 20, 22, 23, 23, ...) 같은 값을 가진 Age_Cate1은 일부 값이 3개 이상의 동일 값 표본을 갖기 때문에 **범주형 확률변수(Categorical R.V.)**로 취급될 가능성이 높다
- 자유도 요건은 범주화가 통계적으로 유의미한지를 결정한다

### 3.4 표 작성 (Tabulation)

표 작성(Tabulation)은 분류된 데이터를 행과 열에 체계적으로 배열하여 **비정형 데이터를 정형 데이터로 변환**하는 과정이다.

표 작성 과정은 다음 단계를 따른다:

1. **실험 설계** — 연구 질문 정의
2. **모집단 결정** — 대상 모집단 식별
3. **변수 설정 / 값 획득** — 특성 정의 및 측정값 수집
4. **사전 검사 (Prescreening)** — 데이터 정제 및 검증 (데이터 마이닝)

### 3.5 특성 추출, 선택, 엔지니어링

- **특성 추출 (Feature Extraction)**: 표로 정리된 원시 데이터에서 의미 있는 변수를 도출
- **특성 선택 (Feature Selection)**: 분석에 유용한 정보를 제공하지 않는 저신호 변수를 제거
- **특성 엔지니어링 (Feature Engineering)**: 모델 성능을 향상시키기 위해 도메인 관련의 유익한 속성을 도입

**구체적인 특성 추출 예시** — 비정형 문자 시퀀스(AAAAA / BB / CCC / D)의 표 작성:

| Sample | # of Alphabets | # of A | # of B | # of C | # of D |
|--------|---------------|--------|--------|--------|--------|
| S1 | 5 | 5 | 0 | 0 | 0 |
| S2 | 5 | 5 | 0 | 0 | 0 |
| S3 | 5 | 5 | 0 | 0 | 0 |
| S4 | 2 | 0 | 2 | 0 | 0 |
| S5 | 2 | 0 | 2 | 0 | 0 |
| S6 | 2 | 0 | 2 | 0 | 0 |
| S7 | 3 | 0 | 0 | 3 | 0 |
| S8 | 3 | 0 | 0 | 3 | 0 |
| S9 | 3 | 0 | 0 | 3 | 0 |
| S10 | 1 | 0 | 0 | 0 | 1 |
| S11 | 1 | 0 | 0 | 0 | 1 |
| S12 | 1 | 0 | 0 | 0 | 1 |

이 기법은 이미지, 비디오, 시퀀스 및 기타 비정형 형식에 적용할 수 있다.

### 3.6 데이터셋의 유형

**레코드 기반 데이터:**
- **데이터 행렬 (Data Matrix)**: n x p 행렬로 n개의 행 = 객체, p개의 열 = 속성
- **문서 데이터 (Document Data)**: 각 문서가 용어 벡터(단어 빈도)로 표현됨
- **트랜잭션 데이터 (Transaction Data)**: 각 레코드가 항목의 집합 (장바구니 데이터)

**그래프 기반 데이터:**
- 월드 와이드 웹, 분자 구조, 지도 데이터

**순서 데이터:**
- 공간 데이터, 시간 데이터, 순차 데이터, 유전/유전체 시퀀스 데이터

> **핵심 포인트:** 정형 데이터 = 확률변수의 집합이며, 데이터 = 변수의 조합이다. 분석의 질은 근본적으로 원시 데이터를 얼마나 잘 표로 만들고 구조화하는지에 달려 있다.

## 4. 지도 학습 vs 비지도 학습 개요

### 4.1 비교표

| 측면 | 지도 학습 (Supervised Learning) | 비지도 학습 (Unsupervised Learning) |
|------|--------------------------------|-------------------------------------|
| **정답** | 사전 정의된 정답(레이블)이 존재 | 사전 정의된 정답 없음 |
| **정답의 수** | 단일 정답 | 다수의 가능한 정답 |
| **목표** | 예측 (데이터와 클래스 간의 관계 발견) | 패턴 인식 (데이터 구조 또는 숨겨진 패턴 발견) |
| **예시 작업** | 분류(Classification), 회귀(Regression) | 군집화(Clustering), 연관(Association) |
| **레이블** | 있음 | 없음 |
| **다양한 결과** | 아니오 | 예 |
| **학습 데이터** | 레이블 있음 (입력-출력 쌍) | 레이블 없음 (입력만) |

**지도 학습(SL)과 비지도 학습(USL) 모두에 필요한 데이터 요건:**

| | 정형 데이터 | 사전 정의된 정답 (= 클래스) |
|---|---|---|
| **지도 학습** | 필요 | 필요 |
| **비지도 학습** | 필요 | 불필요 |

지도 학습과 비지도 학습 모두 **정형 데이터**가 필요하며, 차이점은 **사전 정의된 정답(클래스)**의 존재 여부이다.

### 4.2 지도 학습에서의 클래스 (레이블)

"클래스(class)"에는 여러 동의어가 있다: **레이블(label), 정답(exact answer), 실측값(ground truth), 반응 변수(responsible variable), 결과 변수(outcome variable), 종속 변수(dependent variable)**. 중요한 것은, **어떤 확률변수든 클래스가 될 수 있다** — 이는 분석 목적에 따라 달라진다.

> **Q: 정형 데이터에 "클래스"가 항상 존재하는가?**
>
> **A:** 이는 **분석의 목적**에 의해 결정된다. 클래스의 존재는 데이터 자체의 고유한 속성이 아니라, 사용자가 무엇을 원하는지에 달려 있다. 특정 변수를 예측하거나 추론하고 싶을 때 해당 변수를 "클래스"로 지정한다 (지도 학습 수행). 예측 없이 구조나 패턴을 발견하고 싶을 때는 "클래스" 없이 작업한다 (비지도 학습 수행).

### 4.3 분류와 회귀 (지도 학습)

- **분류 (Classification)**: 알려진 레이블이 있는 학습 집합을 기반으로 새로운 관측치를 사전 정의된 범주에 할당; 새로운 데이터에 대해 **범주형 레이블**을 예측
- **회귀 (Regression)**: 새로운 데이터에 대해 **연속형 값**을 예측 (예: 데이터 포인트를 통과하는 직선 적합)

### 4.4 군집화와 연관 (비지도 학습)

비지도 학습은 종종 **인간 두뇌의 지능**에 비유되며, **패턴 인식**이 그 핵심 메커니즘이다. 인간은 자연스럽게 주변 세계를 그룹화하고 분류한다 — 이는 타고난 능력이다. 0과 1만 처리하는 컴퓨터는 이를 복제하기 위해 특별히 설계된 함수가 필요하다.

- **군집화 (Clustering, 클래스 발견)**: 객체 간의 **유사성**을 기반으로 객체를 부분집합(클러스터)으로 그룹화; 구조를 발견하고 시각화; 이상치 식별과 같은 품질 관리 과정에 활용
- **연관 (Association)**: 레이블 없는 데이터에서 서로 다른 항목 간의 관계를 매핑 (예: 장바구니 분석 — 함께 자주 구매되는 항목 발견)

> **핵심 포인트:** 지도 학습(Supervised Learning)은 레이블이 있는 데이터를 필요로 하며 특정 결과를 예측하는 것이 목표이고 (분류는 범주형, 회귀는 연속형), 비지도 학습(Unsupervised Learning)은 레이블 없는 데이터로 숨겨진 패턴이나 그룹을 발견하는 것이 목표이다 (군집화는 그룹, 연관은 관계).

## 5. 정보학의 응용

정보학은 대부분의 학문 분야와 산업에 적용된다. 응용 범위는 무궁무진하지만, 이 강의에서는 정보학의 개념이 가장 잘 확립된 생의학(bio-medical science) 분야에 초점을 맞춘다.

### 5.1 의료정보학 (Medical Informatics)

의료정보학(Medical Informatics)은 컴퓨터, 통신, 정보 기술 및 시스템을 의학의 모든 분야 — 의료, 의학 교육, 의학 연구에 적용하는 것이다. 의학/생물학, 수학, 정보 시스템, 컴퓨터과학, 통계학, 의사결정 분석, 경제학/보건 정책, 심리학을 통합하는 **다학제적(multidisciplinary)** 분야이다.

**전자의무기록 (Electronic Medical Records, EMR):**

EMR 시스템의 주요 이점:
- **접근성 (Access)**: 기록의 가용성, 전송 및 검색
- **가독성 (Legibility)**: 명확하고 읽기 쉬운 데이터로 추출 및 보고
- **속도 (Speed)**: 종이 기록 대비 **4배 빠른** 데이터 검색
- **데이터 입력 감소**: 데이터 재사용으로 중복 입력 제거
- 구조를 부여함으로써 더 나은 정리
- 저장 공간 효율성
- 집계를 포함한 다중 뷰 허용
- 데이터 입력 시 자동 검사 (철자 검사)
- 데이터 품질 및 표준 강화
- 자동화된 의사결정 지원
- 통계 및 연구 역량

**진단을 위한 의사결정 지원:**

계산 도구가 디지털 영상, 방사선 정보 시스템, 환자 모니터링 시스템을 활용하여 임상의의 진단적 추론 및 치료 계획을 지원한다.

**가상 현실 / 컴퓨터 시뮬레이션:**

**다빈치 수술 시스템 (da Vinci Surgical System)** (Intuitive Surgical, 캘리포니아, 미국)이 대표적인 예이다. 외과의는 수술 영역의 3D 시각화와 로봇 수술 도구를 갖춘 제어 콘솔에 앉아 수술한다.

### 5.2 약학정보학 (Pharmacoinformatics)

약학정보학(Pharmacoinformatics)은 정보학의 원리를 약학에 적용하는 분야이다. 이 역시 대표적인 **다학제적(multidisciplinary)** 분야이다.

- **신약 개발(Drug discovery)**은 화학, 생물학, 약학, 제약 기술 등 여러 과학 및 기술 분야의 통합과 정보 기술의 광범위한 활용이 필요하다
- 핵심 아이디어는 다양한 정보학 분야(생물정보학, 화학정보학, 면역정보학 등)를 하나의 플랫폼으로 통합하여 끊김 없는 신약 개발 과정을 구현하는 것이다
- "Pharmacoinformatics"라는 용어가 처음 언급된 것은 **1993년**이다
- **ASHP의 정보학에서 약사의 역할에 관한 성명**: "건강 결과를 개선하기 위해 약물 사용 과정에서 데이터, 정보, 지식, 기술 및 자동화를 사용하고 통합하는 것"

**약물 사용 과정 주기 (Medication Use Process Cycle):**

선택/조정 --> 처방 --> 검증/조제 --> 투여 --> 모니터링 --> 교육 --> (주기 반복)

| 단계 | 담당자 |
|------|--------|
| **약물 선택 / 조정 (Medication Selection / Reconciliation)** | 처방자, 간호사, 약사 |
| **처방 (Ordering)** | 처방자, 간호사, 약사 |
| **검증 / 조제 (Verification / Dispensing)** | 약사, 기술자 |
| **투여 (Administration)** | 간호사, 보호자, 환자 |
| **모니터링 (Monitoring)** | 처방자, 간호사, 약사, 환자, 보호자, 의료 시스템 |
| **교육 (Education)** | 처방자, 간호사, 약사, 환자, 보호자 |

> **핵심 포인트:** 의료정보학(Medical Informatics)과 약학정보학(Pharmacoinformatics)은 정보학이 도메인 특화 분야와 융합되어 EMR 시스템부터 자동화된 신약 개발 파이프라인까지 데이터 기반 의사결정을 위한 강력한 도구를 만들어내는 방식을 보여준다.

| 개념 | 설명 |
|------|------|
| 융합정보학 (Convergence Informatics) | 데이터 사이언스, 통계학, 프로그래밍, 도메인 지식을 통합하는 다학제적 분야 |
| 데이터 사이언티스트 (Data Scientist) | == 정보학자(Informatician); "21세기의 가장 섹시한 직업"; 수학, 통계, 프로그래밍 (C/C++, Java, R, Python, Julia, SQL, JavaScript), 데이터 마이닝, DB, ML 역량 필요 |
| 데이터 사이언스 범위 | 원칙은 모든 데이터 — 크든 작든 — 에 적용 |
| 모집단 vs 표본 | 모집단 = 모든 가능한 결과; 표본 = 분석에 사용되는 부분집합 |
| 분산 (Variance) | 모분산: N으로 나눔; 표본분산: 비편향 추정을 위해 (n-1)로 나눔 |
| 표본추출 (Sampling) | 오차 = 무작위 변동; 편향 = 체계적 왜곡 |
| 정형 데이터 (Structured Data) | 행/열이 있는 스키마, 모든 실수 값, 행렬 형태 |
| 비정형 데이터 (Unstructured Data) | 전 세계 데이터의 ~85%; 관계형 모델에 맞지 않음; 표 작성을 통해 정형으로 변환 |
| 확률변수 (Random Variables) | 연속형 (수치형, 계산용) vs 범주형 (특성형, 분류용); 이산형 해석은 분석 목적에 따라 달라짐 |
| 자유도 (Freedom Degree) | 범주화에 최소 3개의 동일 값 표본 필요; Age_Cont/Age_Cate1/Age_Cate2 해상도 수준 |
| 표 작성 (Tabulation) | 행과 열에 데이터를 체계적으로 배열; 데이터 사이언스 파이프라인의 핵심 단계 |
| 특성 추출 (Feature Extraction) | 원시 데이터에서 의미 있는 변수를 도출 (예: 비정형 시퀀스에서 문자 빈도 집계) |
| 특성 선택 (Feature Selection) | 저신호 변수를 제거하여 분석 품질 향상 |
| 특성 엔지니어링 (Feature Engineering) | 도메인 관련의 유익한 속성을 도입 |
| 지도 학습 (Supervised Learning) | 레이블 있는 데이터, 단일 정답, 예측 목표; 분류(범주형)와 회귀(연속형) |
| 비지도 학습 (Unsupervised Learning) | 레이블 없는 데이터, 다수 정답, 패턴 인식 목표; 군집화(유사성 그룹화)와 연관(항목 관계) |
| 클래스 (Class, Label) | 어떤 확률변수든 클래스가 될 수 있음; 존재 여부는 데이터 자체가 아닌 분석 목적에 따라 결정 |
| 의료정보학 (Medical Informatics) | EMR (4배 빠른 데이터 접근, 중복 입력 감소), 진단 의사결정 지원, 다빈치 수술 시스템 |
| 약학정보학 (Pharmacoinformatics) | 다학제적 통합으로서의 신약 개발; 1993년 최초 언급; 약물 사용 과정 주기 |
| 생물정보학 (Bioinformatics) | 후반부 강의 초점; 컴퓨터과학, 화학, 공학, 수학, 생화학, 통계학, 생물학의 교차점 |
