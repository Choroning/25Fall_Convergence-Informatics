## 1. R 소개

### 1.1 R이란?

R은 통계 컴퓨팅과 데이터 분석을 위해 설계된 무료 오픈소스 프로그래밍 언어이다. Python과 함께 데이터 사이언스를 위한 가장 대표적인 두 언어 중 하나이다.

### 1.2 데이터 사이언스에 R을 사용하는 이유

- **무료이며 오픈소스**: [https://cran.r-project.org/](https://cran.r-project.org/) 에서 이용 가능
- 실습 중심 데이터 분석에 있어 **비교적 쉽게** 배울 수 있음
- **확장성**: CRAN에서 커뮤니티가 기여한 수천 개의 패키지 이용 가능
- **강력한 통계 지원**: 통계 모델링 및 시각화를 위한 내장 함수 제공

### 1.3 RStudio (R을 위한 IDE)

RStudio는 R 프로그래밍을 보다 쉽게 해주는 통합 개발 환경(IDE, Integrated Development Environment)이다. IDE는 일반적으로 소스 코드 편집기, 빌드 자동화 도구, 디버거로 구성된다.

RStudio는 네 개의 주요 패널로 구성된다:

| 패널 | 용도 |
|------|------|
| 코드 편집기 (Code Editor) | R 스크립트 작성 및 편집 |
| R 콘솔 (R Console) | 대화형으로 명령어 실행 |
| 환경/히스토리 (Environment/History) | 로드된 변수 및 명령어 기록 확인 |
| 파일/플롯/패키지/도움말 (Files/Plots/Packages/Help) | 파일 탐색, 플롯 보기, 패키지 관리, 도움말 접근 |

### 1.4 R 콘솔 / 사용자 프롬프트

- 명령어는 R 사용자 프롬프트(`>`)에서 대화형으로 입력한다
- **위/아래** 방향키로 명령어 기록을 탐색할 수 있다
- 프롬프트 창에서 직접 프로그래밍하는 것은 **권장하지 않는다** — 시스템이 충돌하거나 정전이 발생하면 저장하지 않은 모든 작업을 잃게 된다

### 1.5 좋은 프로그래밍 습관

- 항상 스크립트를 주기적으로 저장하고 백업을 만들 것
- 가장 좋은 습관 중 하나는 스크립트를 수정할 때마다 저장하는 것이다
- 의미 있는 파일 이름 규칙을 사용할 것 — `final`, `Realfinal`, `Realfinal222`와 같은 이름은 피할 것

### 1.6 RStudio 치트시트

RStudio는 **Help > Cheatsheets**를 통해 접근할 수 있는 내장 참조 시트를 제공한다. IDE, 데이터 변환, 시각화 등에 대해 가장 널리 사용되는 명령어와 단축키를 다루고 있다.

> **팁:** 가장 자주 사용하는 연산의 키 바인딩을 외워두면 작업 흐름이 크게 빨라진다.

### 1.7 R 작업 공간 (Workspace)

```r
getwd()       # 현재 작업 디렉토리 확인
setwd("path") # 작업 디렉토리 변경 (주의: "\"는 이스케이프 문자)
```

작업을 저장하는 두 가지 옵션:
1. **R 스크립트 저장** — 코드만 텍스트 형식으로 저장
2. `save.image()`를 사용하여 **R 작업 공간 저장** — RAM에 있는 모든 변수를 저장

> **핵심 포인트:** R은 **대소문자를 구분하는(case-sensitive)** 언어이다. `variableName`과 `variablename`은 서로 다른 두 개의 변수로 취급된다.

## 2. 기본 데이터 유형

### 2.1 핵심 데이터 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| `numeric` | 실수 (숫자의 기본 유형) | `3.14`, `42` |
| `integer` | 정수 (`L`을 붙임) | `5L`, `100L` |
| `character` | 텍스트 문자열 (따옴표로 감쌈) | `"Hello"`, `'World'` |
| `logical` | 논리값 (Boolean) | `TRUE`, `FALSE` |
| `factor` | 수준(level)을 갖는 범주형 변수 | `factor(c("a", "b"))` |

### 2.2 유형 확인 및 검사

```r
class(x)   # 변수의 데이터 유형 확인
str(x)     # 객체의 내부 구조 확인
ls()       # 환경에 로드된 모든 변수 나열
```

### 2.3 데이터 프레임 (Data Frame)

지금까지 정의한 변수들은 단일 숫자에 불과하다. 이는 데이터를 저장하는 데 그리 유용하지 않다. R에서 데이터셋을 저장하는 가장 일반적인 방법은 **데이터 프레임(Data Frame)**이다.

데이터 프레임은 다음과 같은 테이블이다:
- **행(Row)**은 관측치(개별 데이터 포인트)를 나타낸다
- **열(Column)**은 변수(각 관측치에 대해 측정된 속성)를 나타낸다

데이터 프레임은 수치형, 문자형, 논리형, 팩터 등 **서로 다른 데이터 유형을 하나의 객체에 결합**할 수 있기 때문에 특히 유용하다. 이것이 모든 원소가 같은 유형이어야 하는 행렬(Matrix)과의 핵심적인 차이점이다.

```r
# 데이터 프레임 vs. 행렬
# - 데이터 프레임은 열/행 이름과 혼합 데이터 유형(팩터 포함)을 지원
# - 행렬은 모든 원소가 같은 유형이어야 함
```

> **핵심 포인트:** `data()`를 사용하여 내장 데이터셋을 로드하고(예: `data(iris)`), `head()`, `tail()`, `dim()`, `nrow()`, `ncol()`을 사용하여 빠르게 검사할 수 있다.

## 3. 변수와 할당

### 3.1 할당 연산자

```r
x <- 10           # 권장되는 할당 연산자
y <- "Hello"       # 문자열 할당
z <- TRUE          # 논리값 할당
```

`=`도 할당에 사용할 수 있지만, 동등 비교 연산자 `==`와의 혼동을 피하기 위해 `<-`를 사용하는 것이 **강력히 권장**된다.

### 3.2 변수 제거

```r
rm(x)              # 단일 변수 제거
rm(list = ls())    # 환경에서 모든 변수 제거
```

> **핵심 포인트:** R에서 할당에는 항상 `<-`를 사용할 것. `=`는 함수 인수에 사용하도록 한다(예: `rnorm(n = 100, mean = 0)`).

## 4. 벡터와 연산

### 4.1 c()로 벡터 생성

벡터(Vector)는 R에서 가장 기본적인 데이터 구조이다. `c()` 함수는 값을 결합하여 벡터를 생성한다.

```r
numericVector   <- c(1, 2, 3, 4, 5)
characterVector <- c("One", "Two", "Three", "Four", "Five")
logicalVector   <- c(TRUE, TRUE, FALSE, FALSE, FALSE)
```

### 4.2 수열 생성

```r
# 콜론 연산자 사용
1:10                              # 1, 2, 3, ..., 10

# seq() 사용
seq(from = 1, to = 10)            # 1:10과 동일
seq(from = 1, to = 10, by = 2)    # 1, 3, 5, 7, 9
seq(1, 10, length = 5)            # 균등 간격의 5개 값

# rep() 사용
rep(1, 3)                         # 1, 1, 1
rep(c("a", "b"), 5)               # "a","b","a","b",...
rep(1:3, each = 10)               # 1,1,...,2,2,...,3,3,...
```

### 4.3 난수 생성

```r
rnorm(n = 100, mean = 180, sd = 1)  # N(180, 1)에서 100개의 난수 값
```

### 4.4 벡터 인덱싱

```r
x <- c(10, 20, 30, 40, 50)
x[1]          # 첫 번째 원소: 10
x[c(1, 3)]   # 첫 번째와 세 번째 원소: 10, 30
x[-2]         # 두 번째를 제외한 모든 원소: 10, 30, 40, 50
```

### 4.5 벡터 산술 연산

```r
a <- c(1, 2, 3)
b <- c(4, 5, 6)

a + b    # 원소별 덧셈: 5, 7, 9
a * b    # 원소별 곱셈: 4, 10, 18
a > 2    # 원소별 비교: FALSE, FALSE, TRUE
```

### 4.6 벡터 결합

```r
cbind(a, b)   # 열 방향으로 결합 (나란히)
rbind(a, b)   # 행 방향으로 결합 (쌓기)
```

> **핵심 포인트:** R에서 벡터에 대한 연산은 기본적으로 **벡터화(vectorized)**되어 있어, 명시적인 반복문 없이 원소별로 적용된다.

## 5. 기본 함수

### 5.1 출력 함수

```r
print("Hello World")     # 단일 객체의 서식화된 출력
cat("Value:", x, "\n")   # 연결된 일반 텍스트 출력
```

### 5.2 요약 통계 함수

```r
sum(1:100)       # 합: 5050
mean(x)          # 산술 평균
median(x)        # 중앙값
sd(x)            # 표준편차
length(x)        # 원소 개수
min(x)           # 최솟값
max(x)           # 최댓값
summary(x)       # 최솟값, Q1, 중앙값, 평균, Q3, 최댓값
```

### 5.3 데이터 검사 함수

```r
head(iris)          # 처음 6행 (기본값)
tail(iris)          # 마지막 6행 (기본값)
head(iris, n = 10)  # 처음 10행
dim(iris)           # 차원 (행, 열)
nrow(iris)          # 행 수
ncol(iris)          # 열 수
fix(iris)           # 스프레드시트 형태의 편집기에서 데이터 열기
```

### 5.4 행렬과 팩터 함수

```r
# 행렬 생성
A <- matrix(NA, nrow = 10, ncol = 5)

# 팩터 변수 생성
B <- factor(c("a", "b", "a", "b"))
levels(B)     # 팩터 수준 확인
```

### 5.5 도움말 사용

```r
help(sum)      # 함수의 도움말 페이지 열기
?sum           # help()의 단축키
help.start()   # 브라우저에서 R의 HTML 도움말 시스템 열기
```

> **핵심 포인트:** `summary()`를 수치형 변수에 사용하면 기술 통계량을 얻을 수 있고, 팩터 변수에 사용하면 빈도수를 얻을 수 있다. 동작이 변수 유형에 따라 자동으로 달라진다.

## 6. 패키지

### 6.1 패키지 설치 및 로드

```r
# 패키지 다운로드를 위한 저장소 설정
setRepositories()

# CRAN에서 패키지 설치
install.packages("ggplot2")

# 현재 세션에 패키지 로드
library(ggplot2)
```

### 6.2 패키지 찾기

- 이용 가능한 패키지 탐색: [https://cran.r-project.org/web/packages/available_packages_by_name.html](https://cran.r-project.org/web/packages/available_packages_by_name.html)
- 특정 기능에 대해 Google 검색 (예: "R package for boxplot")

### 6.3 패키지를 활용한 시각화

R은 기본적인 플로팅 함수를 제공하지만, `ggplot2`나 `ggpubr` 같은 서드파티 패키지를 사용하면 보다 고급스럽고 시각적으로 우수한 시각화가 가능하다.

```r
# 기본 R 시각화
hist(HeightMale)                                          # 히스토그램
boxplot(Height ~ Gender, data = ourData, col = c("blue", "red"))  # 상자 그림

# 고급 시각화는 ggplot2 / ggpubr 패키지 필요
```

> **핵심 포인트:** R의 가장 큰 강점 중 하나는 패키지 생태계이다. 새로운 함수를 작성하고 이를 "R 패키지"(또는 "R 라이브러리")로 패키징하여 CRAN을 통해 커뮤니티에 공유할 수 있다.
