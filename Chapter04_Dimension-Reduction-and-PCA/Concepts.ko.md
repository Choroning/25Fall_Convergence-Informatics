## 1. 차원 축소

### 1.1 정의

차원 축소(Dimensionality Reduction)는 수천 또는 수백만 개의 저수준 특성에서 가장 관련성 높은 특성을 선택하여 **더 좋고(Better)**, **더 빠르며(Faster)**, **더 이해하기 쉬운(Easier to understand)** 학습 모델을 구축하는 과정이다.

### 1.2 차원 축소가 필요한 이유

- **학습 알고리즘 속도 향상**: 고차원에서 비핵심 변수는 성능 저하를 초래하거나 처리해야 할 데이터 양이 극도로 커질 수 있음
- **더 단순하고 안정적인 모델**: 너무 많은 차원으로 학습된 모델은 과적합(Overfit)되기 쉬움; 차원을 줄이면 더 강건해짐
- **더 나은 시각화**: 저차원은 데이터 구조를 시각화하기 쉽게 만듦

### 1.3 차원의 저주(Curse of Dimensionality)

차원 수가 증가하면 공간은 기하급수적으로 커지고, 동일한 양의 데이터 밀도는 희박해진다:

| Dimensions | Space | Data Points | Density |
|-----------|-------|-------------|---------|
| 1 | 10 | 8 | 80% |
| 2 | 100 | 8 | 8% |
| 3 | 1000 | 8 | 0.8% |

### 1.4 응용 분야

차원 축소는 다양한 분야에서 널리 사용된다:
- 생물정보학(Bioinformatics)
- 머신 비전(Machine Vision)
- 텍스트 분류(Text Categorization)
- 시장 분석(Market Analysis)
- 품질 관리(Quality Control)
- OCR / 필기 인식(Handwriting Recognition)

> **핵심 포인트:** 오컴의 면도날(Occam's Razor)이 적용된다 -- "동등하게 좋은 두 가설에 직면했을 때, 항상 더 단순한 것을 선택하라." 차원을 줄이면 종종 더 단순하고 일반화 가능한 모델이 만들어진다.

## 2. 특성 선택 vs. 차원 축소

### 2.1 특성 선택(Feature Selection)

- 새로운 특성은 원래 특성의 **부분집합**
- 분류를 위해 소수의 특성만 계산하면 됨
- 원래 특성의 해석 가능성을 보존

### 2.2 차원 축소 (특성 추출, Feature Extraction)

- 새로운 특성은 원래 특성의 **조합** (PCA/LDA의 경우 선형 조합)
- 합성적인 복합 변수를 생성
- 해석이 어려워지지만, 더 적은 차원에서 더 많은 정보를 포착할 수 있음

> **핵심 포인트:** 특성 선택은 원래 특성을 유지한다 (해석 가능). 차원 축소는 새로운 결합 특성을 생성한다 (더 압축적이지만 해석이 어려움).

## 3. 두 가지 주요 접근법

### 3.1 투영(Projection)

고차원 데이터를 가능한 한 많은 정보를 보존하면서 저차원 부분공간에 투영한다. 예를 들어, 3D 데이터를 2D 평면에 투영하는 것이다.

### 3.2 매니폴드 학습(Manifold Learning)

- **매니폴드(Manifold)**는 고차원 데이터를 포함하는 부분공간이다
- **매니폴드 학습**은 고차원 데이터에 내재된 저차원 구조를 발견한다
- 단계: (1) 데이터 구조를 그래프로 표현, (2) 저차원 임베딩 계산, (3) ML 적용

**동기 부여 예제:**

- **스위스 롤(Swiss Roll)**: 데이터가 3D로 말린 2D 표면 위에 놓여 있다. 점 A1과 A2는 유클리드 거리(롤을 관통하는 직선)로는 멀리 떨어져 있을 수 있지만, 매니폴드 표면을 따라가면 가깝다. 주변 공간에서의 유클리드 거리는 매니폴드 위의 실제 근접성을 포착하지 못한다.
- **골프 스윙(Golf Swing)**: 골프 스윙의 순차적 프레임들은 고차원 픽셀 공간에서 매니폴드를 형성한다. 유클리드 거리를 사용하면 특정 프레임의 최근접 이웃에 시각적으로 다른 자세가 포함될 수 있다. 매니폴드를 발견한 후에는 의미적으로 가까운 데이터(시간적으로 인접한 스윙 위치)가 올바르게 이웃으로 식별된다.

### 3.3 방법 분류

```
Dimensionality Reduction
├── Linear
│   └── PCA
└── Non-linear
    ├── Preserve global structure
    │   ├── MDS
    │   └── t-SNE
    └── Preserve local structure
        └── UMAP
```

> **핵심 포인트:** PCA와 같은 선형 방법은 데이터가 선형 부분공간 근처에 있을 때 잘 작동한다. 비선형 방법(MDS, t-SNE, UMAP)은 더 복잡한 구조를 포착한다.

## 4. 선형대수 사전 지식

### 4.1 벡터(Vectors)

- 데이터는 벡터로 기술할 수 있다 (물리학, 컴퓨터 과학, 수학의 관점)
- **벡터의 상수배**: 스칼라로 벡터를 확장
- **벡터의 합**: 두 벡터를 성분별로 더함
- **행 벡터 vs. 열 벡터**: 행렬 곱셈은 행 벡터와 열 벡터의 내적으로부터 따름

### 4.2 선형 결합(Linear Combination)

선형 결합은 벡터의 상수배와 합을 결합한 것으로 -- 두 가지 (또는 그 이상의) 특성을 동시에 결합하여 벡터를 기술한다.

### 4.3 선형 변환(Linear Transformation)

- **선형 결합을 보존하는** 두 벡터 공간 사이의 함수
- **행렬은 곧 선형 변환**: 행렬 A를 벡터 x에 적용하면 다른 벡터 Ax가 생성됨
- 고유벡터와 고유값이 여기서 발생: Ax = lambda * x는 행렬이 고유벡터 방향으로 (lambda만큼만) 스케일링함을 의미

### 4.4 PCA와의 연결

데이터가 고차원 벡터 공간에서 표현될 때, 공분산 행렬은 선형 변환으로 작용한다. 그 고유벡터를 찾으면 데이터가 가장 넓게 퍼진 방향(주성분)이 드러난다.

## 5. Principal Component Analysis (PCA, 주성분 분석)

### 5.1 개요

PCA는 아마도 "표준" 다변량 방법 중 가장 널리 사용되고 잘 알려진 방법일 것이다. Pearson(1901)과 Hotelling(1933)에 의해 발명되었으며, Goodall(1954)에 의해 "인자 분석(Factor Analysis)"이라는 이름으로 생태학에 처음 적용되었다.

PCA는 상관관계가 있을 수 있는 n개 객체 x p개 변수의 데이터 행렬을 받아, 원래 p개 변수의 선형 결합인 **비상관 축(주성분)으로 요약**한다.

### 5.2 PCA의 목표

고차원 공간에서 원본 데이터에 대한 정보를 최대한 보존하는 가장 작은 부분공간을 찾는 것이다.

- 많은 (p개) 변수를 가진 데이터를 더 적은 (k개) 파생 변수 집합으로 요약
- **첫 k개 성분**은 객체 간 변동을 가능한 한 많이 표시
- 표현의 명확성과 지나친 단순화(정보 손실) 사이의 균형
- "잔차" 변동 = 축소된 표현에 유지되지 않은 정보

### 5.3 PCA는 데이터를 투영한다

PCA는 원본 데이터의 구조를 유지하면서 벡터에 데이터를 투영하여 차원을 축소한다. **가장 큰 분산**을 가진 축에 먼저 데이터를 투영한다.

> **핵심 포인트:** PCA는 공분산 행렬에 의해서만 영향을 받는다. 데이터에서 최대 분산의 방향을 찾아 그 방향으로 데이터를 투영한다.

## 6. PCA 이론 및 유도

### 6.1 수학적 설정

**X** = [x_1, ..., x_m]을 N x m 데이터 행렬(N차원, m개 인스턴스)이라 하자.

**U**를 N x N 직교 행렬이라 하자, 여기서 **UU^T = I_N**.

- 변환: **y = Ux**
- 역변환: **x = U^T y**
- M개 기저 벡터를 사용한 근사 (M <= N): **x_hat = sum(u_i * y_i) for i=1 to M**

### 6.2 재구성 오차(Reconstruction Error)

$$\varepsilon^2 = \mathbb{E}\{\|\mathbf{x} - \hat{\mathbf{x}}\|^2\} = \frac{1}{m} \sum_{j=1}^{m} \|\mathbf{x}_j - \hat{\mathbf{x}}_j\|$$

**목표:** 재구성 오차를 최소화:

$$\arg \min_{\mathbf{U}} \varepsilon^2, \quad \text{s.t.} \quad \mathbf{U}^T \mathbf{U} = \mathbf{I}_N$$

### 6.3 라그랑주 승수법을 통한 유도

오차는 다음과 같이 표현할 수 있다:

$$\varepsilon^2 = \sum_{i=M+1}^{N} \mathbf{u}_i^T \boldsymbol{\Sigma} \mathbf{u}_i$$

여기서 **Sigma**는 공분산 행렬이다 (**x**가 중심화되었으므로: E{xx^T} = Sigma).

직교성 제약에 대해 라그랑주 승수를 사용:

$$L = \sum_{i=M+1}^{N} \mathbf{u}_i^T \boldsymbol{\Sigma} \mathbf{u}_i - \sum_{i=M+1}^{N} \lambda_i(\mathbf{u}_i^T \mathbf{u}_i - 1)$$

도함수를 0으로 놓으면:

$$\frac{\partial L}{\partial \mathbf{u}_i} = 2\boldsymbol{\Sigma}\mathbf{u}_i - 2\lambda_i\mathbf{u}_i = 0 \implies \boldsymbol{\Sigma}\mathbf{u}_i = \lambda_i\mathbf{u}_i$$

### 6.4 고유값 해석

방정식 **Sigma * u_i = lambda_i * u_i**는 다음을 보여준다:
- **u_i**는 공분산 행렬 Sigma의 **고유벡터(Eigenvectors)**
- **lambda_i**는 대응하는 **고유값(Eigenvalues)**

재구성 오차는 다음이 된다:

$$\varepsilon^2 = \sum_{i=M+1}^{N} \lambda_i$$

오차는 lambda_{M+1}, ..., lambda_N이 **가장 작은** 고유값일 때 **최소**가 된다. 이는 **가장 큰** 고유값에 대응하는 고유벡터를 주성분으로 유지함을 의미한다.

### 6.5 공분산 행렬 계산

$$\boldsymbol{\Sigma} = \frac{1}{n} \mathbf{X}^T \mathbf{X}$$

여기서 **X**는 평균 중심화된 데이터 행렬이다. X^T X의 (i,j) 원소는 i번째와 j번째 특성 열의 내적이다.

### 6.6 공분산 행렬의 기하학적 해석

2D 공분산 행렬의 경우:

$$\boldsymbol{\Sigma} = \begin{bmatrix} \sigma_x^2 & \text{Cov}(X,Y) \\ \text{Cov}(X,Y) & \sigma_y^2 \end{bmatrix}$$

- **대각 원소** (sigma_x^2, sigma_y^2): **각 축을 따른 퍼짐(분산)** -- 데이터가 X와 Y를 따라 독립적으로 얼마나 늘어나는지
- **비대각 원소** (Cov(X,Y)): 변수 간의 **결합 퍼짐** -- X와 Y가 함께 어떻게 움직이는지

**X^T X가 공분산을 포착하는 이유:**

X^T X는 **특성 열 간의 내적 행렬**을 생성한다. 각 원소 dot(X_i, X_j)는 모든 관측에 걸쳐 특성 i와 특성 j가 얼마나 정렬되어 있는지를 측정한다. n으로 나누면 이것이 공분산으로 정규화된다. 이것이 X가 평균 중심화되었을 때 Sigma = (1/n) X^T X인 이유이다.

공분산 행렬이 데이터에 대한 선형 변환으로 사용될 때, 데이터를 가장 큰 분산의 방향으로 늘린다. 이 변환의 고유벡터는 그 방향(주성분)을 드러내고, 고유값은 얼마나 많은 늘림이 발생하는지(분산의 양)를 드러낸다.

> **핵심 포인트:** 주성분은 공분산 행렬의 고유벡터이며, 고유값 크기 순으로 정렬된다. 고유값은 각 성분이 설명하는 분산의 양을 나타낸다.

## 7. PCA의 기하학적 근거

### 7.1 기하학적 관점

- 객체는 p개 변수 각각에 대한 축을 가진 다차원 공간에서 n개 점의 구름으로 표현됨
- 점들의 **중심(Centroid)**은 각 변수의 평균으로 정의됨
- 각 변수의 **분산(Variance)**은 평균 주위의 n개 값의 평균 제곱 편차

$$V_i = \frac{1}{n-1} \sum_{m=1}^{n} (X_{im} - \bar{X}_i)^2$$

### 7.2 공분산 행렬에서의 고유값과 고유벡터

| | 원래 의미 | 공분산 행렬에서의 의미 |
|---|---|---|
| **고유벡터** | 행렬이 벡터에 작용하는 주축의 방향을 표현 | 데이터가 분포된 방향을 표현 |
| **고유값** | 고유벡터의 크기 | 증가하는 벡터 공간의 크기 (분산의 양) |

### 7.3 간단한 예제 (2x2 행렬)

A = [[4, 2], [3, 5]]인 경우:

1. 특성 방정식을 풂: det(A - lambda*I) = 0
2. (lambda - 7)(lambda - 2) = 0이므로, lambda = 7, 2
3. lambda = 7일 때: 고유벡터 = c * (2, 3)
4. lambda = 2일 때: 고유벡터 = c * (-1, 1)

> **핵심 포인트:** 고유값은 각 고유벡터 방향을 따라 포착되는 분산의 양을 결정한다. 더 큰 고유값은 더 중요한 주성분에 대응한다.

## 8. 평균, 분산, 공분산

### 8.1 평균 (기대값, Expectation)

|  | 이산형 (PMF) | 연속형 (PDF) |
|--|----------------|-----------------|
| **공식** | E(x) = sum(p_i * x_i) | E(x) = integral(x * f(x) dx) |

### 8.2 분산(Variance)

|  | 이산형 (PMF) | 연속형 (PDF) |
|--|----------------|-----------------|
| **공식** | V(X) = sum((x_i - m)^2 * p_i) = E(X^2) - m^2 | Var[X] = integral((x - E[X])^2 * f(x) dx) = E[X^2] - E[X]^2 |

표준편차: sigma(X) = sqrt(V(X))

### 8.3 공분산(Covariance)

확률 변수 X와 Y에 대해 E(X) = mu, E(Y) = nu일 때:

$$\text{Cov}(X, Y) = E((X - \mu)(Y - \nu)) = E(XY) - \mu\nu = E(XY) - E(X)E(Y)$$

**X와 Y가 독립이면**, E(XY) = E(X)E(Y) = mu * nu이므로, **Cov(X, Y) = 0**이다.

이것이 PCA가 **비상관(Uncorrelated)** 주성분을 생성하는 이유이다: 공분산 행렬의 고유벡터는 임의의 두 성분 간 공분산이 0인 새로운 축을 정의한다.

### 8.4 상관계수(Correlation)

공분산은 변수의 절대적 스케일에 영향을 받으므로, 정규화한다:

$$\rho = \frac{\text{Cov}(X, Y)}{\sqrt{\text{Var}(X) \cdot \text{Var}(Y)}}, \quad -1 \leq \rho \leq 1$$

## 9. 기타 차원 축소 방법

### 9.1 MDS (Multi-Dimensional Scaling, 다차원 척도법)

- **비선형** 방법으로 **전역 구조(Global Structure)**를 보존
- 저차원 공간에서 데이터 포인트 간의 쌍별 거리를 보존하려고 시도
- 두 가지 변형: **Metric MDS** (= Principal Coordinate Analysis, PCoA)와 **Non-Metric MDS**
- **PCA vs. PCoA**: PCA는 "공분산"에 기반하여 차원을 축소하고, PCoA (Metric MDS)는 "거리"에 기반하여 차원을 축소
- 유클리드 거리를 사용할 때, PCoA (MDS)와 PCA는 **동일한** 차원 축소 결과를 생성 (거리를 최소화한 군집화 = 상관관계를 최대화한 군집화)
- 거리 척도에는 다음이 포함: (1) **유클리드 거리** (피타고라스 정리), (2) **로그 배수 변화의 절댓값** (비대칭 차이를 대칭적으로 만들기 위해 로그를 사용한 후, 표본 간 절댓값의 평균을 취함)

### 9.2 t-SNE (t-distributed Stochastic Neighbor Embedding)

- **비선형** 벡터 시각화 알고리즘으로 **전역 구조(Global Structure)**를 보존
- 고차원에서의 점 간 **유사성**을 보존하면서 차원 축소를 수행하여 임베딩 공간에 배치
- **왜 단순 투영만으로는 안 되는가?** 1D로의 단순 투영은 군집 분리를 파괴할 수 있음 (예: 2D에서 잘 분리된 두 군집이 단일 축에 투영되면 겹칠 수 있음)

**알고리즘 개요:**

1. 각 관심 점 x_i에 대해, 모든 다른 점 x_j까지의 거리를 계산
2. Perplexity sigma_i로 나눔 (안정성을 더하는 매개변수; 각 점마다 다른 값을 가짐)
3. 음의 지수 함수를 적용하여 조건부 확률 p_{j|i}를 구함 (가까운 점일수록 높은 확률)
4. 대칭화: p_ij = (p_{j|i} + p_{i|j}) / 2n
5. 임베딩 공간에서 **t-분포** (가우시안이 아님)를 사용하여 유사도 q_ij를 계산: q_ij = (1 + |y_i - y_j|^2)^{-1} / sum
6. 임베딩 위치를 조정하여 p_ij와 q_ij 간의 KL-발산(KL-divergence)을 최소화

**t-분포를 사용하는 이유?** t-분포는 정규분포보다 꼬리가 더 무거운(Heavier Tails) 특성을 가지며, 이는 평균으로부터 더 넓게 분포되어 있음을 의미한다. 이로 인해 먼 점들이 임베딩에서 더 많은 여유 공간을 갖게 되어 군집 분리가 향상된다.

**핵심 특성:** 데이터의 이동 위치가 계산할 때마다 변하기 때문에 매번 결과가 달라진다. 시각화에는 유용하지만 결정적(Deterministic)이지 않다.

### 9.3 UMAP (Uniform Manifold Approximation and Projection)

- **비선형** 방법으로 **지역 구조(Local Structure)**를 보존
- **위상적 데이터 분석(Topological Data Analysis)**과 **매니폴드 학습(Manifold Learning)**에 기반
- 위상수학의 개념을 사용: 단체(Simplices) (0-simplex = 점, 1-simplex = 변, 2-simplex = 삼각형, 3-simplex = 사면체), 단체 복합체(Simplicial Complexes), 신경 정리(Nerve Theorem)
- 다양한 반경의 원으로 데이터를 덮고 단체 복합체를 구축하여 데이터의 중요한 위상을 포착

**UMAP vs. t-SNE:**

| UMAP | t-SNE |
|------|-------|
| 임베딩 차원의 크기에 제한 없음 | 임베딩이 2차원 또는 3차원에서만 가능 |
| **지역 이웃 그래프(Local Neighbor Graph)** 구조를 더 잘 보존 | **전체(Overall)** 구조를 더 잘 보존 |
| t-SNE보다 **빠름** | 시간이 perplexity라는 하이퍼파라미터에 의해 결정됨 |

### 9.4 비교

| 방법 | 유형 | 보존 대상 | 속도 | 사용 사례 |
|--------|------|-----------|-------|----------|
| PCA | 선형 | 전역 분산 | 빠름 | 일반 축소, 전처리 |
| MDS | 비선형 | 전역 거리 | 보통 | 거리 행렬이 있을 때 |
| t-SNE | 비선형 | 전역 구조 | 느림 | 군집의 시각화 |
| UMAP | 비선형 | 지역 구조 | 빠름 | 시각화, 위상 보존 |

> **핵심 포인트:** PCA는 선형 데이터 및 전처리에 가장 적합하다. 복잡한 구조를 가진 고차원 데이터의 시각화에는 t-SNE와 UMAP이 선호된다.

## 10. 실용적 고려사항

### 10.1 스크리 도표(Scree Plot)

스크리 도표는 고유값(또는 설명 분산 비율)을 내림차순으로 표시한다. 유지할 성분 수를 결정하는 데 도움이 된다:
- 곡선이 평탄해지는 "팔꿈치(Elbow)" 지점을 찾음
- 팔꿈치 이전의 성분들이 가장 의미 있는 분산을 포착

**고유값 분해와의 연결:** 스크리 도표에 그려지는 값은 PCA 고유값 분해(Sigma * u_i = lambda_i * u_i)에서 나온 고유값 **그 자체**이다. 각 고유값 lambda_i는 i번째 주성분이 포착하는 분산을 나타낸다. 스크리 도표는 단순히 이 고유값들을 내림차순으로 배열하여 더 많은 성분을 추가하는 것의 수확 체감(Diminishing Returns)을 시각화한다.

### 10.2 설명 분산 비율(Explained Variance Ratio)

k번째 성분이 설명하는 전체 분산의 비율:

$$\text{Explained Variance Ratio}_k = \frac{\lambda_k}{\sum_{i=1}^{N} \lambda_i}$$

일반적인 경험 법칙은 전체 분산의 80-95%를 설명하기에 충분한 성분을 유지하는 것이다.

### 10.3 언제 어떤 방법을 사용할 것인가

| 시나리오 | 권장 방법 |
|----------|--------------------|
| 선형 관계, 전처리 | PCA |
| 해석 가능한 특성이 필요 | 특성 선택(Feature Selection) |
| 군집의 2D/3D 시각화 | t-SNE 또는 UMAP |
| 쌍별 거리 보존 | MDS |
| 대규모 데이터셋, 빠른 시각화 | UMAP |

> **핵심 포인트:** PCA를 적용하기 전에 항상 데이터를 중심화(Center)하고 (대개 스케일링도) 해야 한다. 방법 간의 선택은 데이터 구조가 선형인지 비선형인지, 목표가 분석인지 시각화인지에 따라 달라진다.
