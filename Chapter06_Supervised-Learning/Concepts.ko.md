## 1. 분류의 정의

### 1.1 군집화 vs. 분류

| 측면 | 군집화 (Clustering) | 분류 (Classification) |
|------|-------------------|---------------------|
| 학습 유형 | **비지도 (Unsupervised)** 학습 | **지도 (Supervised)** 학습 |
| 클래스 | 사전에 알려지지 않음 | **사전 정의됨** |
| 목표 | 클래스 **발견** (새로운/미지의 클래스 식별) | 클래스 **예측** (알려진 클래스에 할당) |
| 성격 | 탐색적 분석 | 예측적 분석 |

- **군집화 (Clustering)**는 클래스 변수(범주형 반응 변수)를 생성하는 과정이다
- **분류 (Classification)**는 클래스 변수를 가진 모형을 수립하고, 그 모형으로부터 클래스 변수를 예측하는 과정이다

> **핵심 포인트:** 군집화는 데이터에서 알려지지 않은 그룹을 발견하고(비지도), 분류는 사전 정의된 그룹에 대한 소속을 예측한다(지도).

## 2. 분류 vs. 예측

### 2.1 분류 (Classification)

- 훈련 세트를 사용하여 구축된 모형을 기반으로 데이터를 분류
- **범주형 클래스**의 값을 분류 속성으로 사용
- 새로운 데이터를 사전 정의된 범주에 할당

### 2.2 예측 (Prediction)

- 훈련 세트를 사용하여 구축된 모형을 기반으로 데이터를 예측
- 클래스 변수의 **유형에 관계없이** 작동 (연속형, 범주형, 생존 시간 등)

> **핵심 포인트:** 결과 변수의 유형에 따라, 분류와 예측은 본질적으로 동일한 과정이다. 분류는 구체적으로 범주형 결과를 지칭하며, 예측은 보다 일반적인 용어이다.

## 3. 분류 실습 예제

**교수 재직 데이터셋 (Faculty Tenure Dataset):**

| NAME | RANK | YEARS | TENURED |
|------|------|-------|---------|
| Mike | Assistant Prof | 3 | No |
| Mary | Associate Prof | 7 | Yes |
| Bill | Professor | 2 | Yes |
| Jim | Assistant Prof | 7 | Yes |
| Dave | Associate Prof | 6 | No |
| Anne | Associate Prof | 3 | No |
| Jeff | Professor | 4 | ? |

**추출된 규칙:** IF rank = 'Professor' OR years > 6 THEN tenured = 'Yes'

**Jeff 분류:** Jeff는 4년 경력의 Professor이다. rank = 'Professor'이므로 규칙이 발동되어, Jeff는 **"Yes" (재직)**로 분류된다.

이 예제는 분류의 전체 파이프라인을 보여준다: 알려진 레이블이 있는 훈련 데이터에서 규칙(모형)이 생성되고, 이 규칙이 새로운 미지의 인스턴스를 분류하는 데 적용된다.

## 4. 일반적인 분류 절차

분류 분석은 구조화된 파이프라인을 따른다:

```
Data
  |
  v
Pre-processing
  - Standardization
  - Normalization
  - Transformation
  |
  v
Classifier Construction  <---->  Tuning Parameters (feedback loop)
  - Feature selection
  - Algorithm
  - Training data
  |
  v
Classifier Evaluation
  - Error rate
  - Testing data
  |
  v
Class Prediction
  - New case
```

### 4.1 2단계 프로세스

**1단계: 모델 구축 (Model Construction)**
- 사전 정의된 클래스 집합을 기술
- 각 튜플/샘플은 클래스 레이블 속성에 의해 결정된 사전 정의된 클래스에 속한다고 가정
- 모델 구축에 사용되는 튜플 집합: **훈련 세트 (Training set)**
- 모델은 분류 규칙, 의사 결정 트리 또는 수학적 공식으로 표현

**2단계: 모델 사용 (Model Usage)**
- 미래의 또는 미지의 객체를 분류하기 위해 사용
- 테스트 샘플의 알려진 레이블을 모형으로부터의 분류 결과와 비교
- **정확도 (Accuracy rate)** = 모형에 의해 올바르게 분류된 테스트 세트 샘플의 비율
- 테스트 세트는 훈련 세트와 **독립적**이어야 하며, 그렇지 않으면 과적합(Overfitting)이 발생

> **핵심 포인트:** 모델은 먼저 훈련 데이터로부터 구축(귀납)된 다음, 새로운 미지의 데이터를 분류하기 위해 적용(연역)된다. 신뢰성 있는 평가를 위해 훈련 데이터와 테스트 데이터는 반드시 분리되어야 한다.

## 5. 훈련 세트와 테스트 세트

### 5.1 정의

| 세트 | 목적 |
|------|------|
| **훈련 세트 (Training set)** | 분류 모형(분류기)을 구축하기 위해**만** 사용되는 데이터 |
| **테스트 세트 (Test set)** (검증 세트, Validation set) | 분류 모형의 성능을 평가하기 위해**만** 사용되는 데이터 |

### 5.2 귀납과 연역

1. **귀납 (Induction)**: 학습 알고리즘이 훈련 세트를 처리하여 학습하고 모형을 생성
2. **연역 (Deduction)**: 학습된 모형이 테스트 세트에 적용되어 예측을 수행

> **핵심 포인트:** 훈련 세트와 테스트 세트를 구분하지 않고 데이터를 사용하면, 모형이 과적합(Overfitting)될 가능성이 높다 — 알려진 데이터에서는 잘 작동하지만 새로운 미지의 데이터에서는 성능이 저하된다.

## 6. 과적합

### 6.1 정의

통계학에서 과적합(Overfitting)이란 *"특정 데이터 세트에 너무 가깝거나 정확하게 대응하는 분석을 생성하여, 추가 데이터에 적합하거나 미래 관측을 신뢰성 있게 예측하는 데 실패할 수 있는 것"*이다.

- 과적합된 모형은 데이터로 정당화할 수 있는 것보다 더 많은 모수를 포함한다 (**과매개변수화, Overparameterization**)
- 머신러닝에서 이 현상을 **과훈련 (Overtraining)**이라고도 한다

### 6.2 과적합이 발생하는 이유

- 훈련 데이터는 입력에서 출력으로의 매핑에 대한 규칙성 정보를 포함하지만, **노이즈(Noise)**도 포함한다
- 목표값이 신뢰할 수 없을 수 있다
- **표본 오차 (Sampling error)**: 선택된 특정 훈련 사례들 때문에 우연히 발생하는 규칙성
- 모형을 적합할 때, 어떤 규칙성이 실제이고 어떤 것이 표본 오차에 의한 것인지 구분할 수 없으므로 두 종류 모두를 적합
- 모형이 매우 유연하면 표본 오차를 아주 잘 모형화할 수 있다 — 이것은 재앙이다

### 6.3 과적합 vs. 과소적합

| 문제 | 설명 |
|------|------|
| **과적합 (Overfitting)** | 훈련 세트에 지나치게 적합하여 실제 새 데이터에 대한 예측 성능이 저하 |
| **과소적합 (Underfitting)** | 모형이 너무 단순; 훈련 오차와 테스트 오차 모두 큼 |

### 6.4 회귀에서의 과적합

세 가지 적합 접근법 비교:

| 방법 | 훈련 데이터 적합도 | 일반화 (Generalization) |
|------|-------------------|----------------------|
| **선형 (Linear)** (y = beta_0 + beta_1 * x) | 보통 | 양호 |
| **2차 (Quadratic)** (y = beta_0 + beta_1 * x + beta_2 * x^2) | 더 좋음 | 더 좋음 |
| **구간별 선형 (Piecewise linear)** (비모수) | 완벽 | 나쁨 (과적합) |

- 핵심 질문은 어떤 방법이 데이터에 가장 잘 적합하느냐가 아니라: **"같은 분포에서 추출된 미래 데이터를 얼마나 잘 예측할 것인가?"**이다
- 테스트 세트 방법: 데이터의 약 30%를 무작위로 테스트 세트로 선택하고, 나머지로 훈련한 후, 테스트 세트에서 평균 제곱 오차(MSE)로 평가

### 6.5 구체적인 MSE 비교 값

**테스트 세트 방법:**

| Model | MSE |
|-------|-----|
| Linear | 2.4 |
| Quadratic | 0.9 |
| Piecewise | 3.2 |

**LOOCV 방법:**

| Model | MSE |
|-------|-----|
| Linear | 2.12 |
| Quadratic | 0.962 |
| Piecewise | 3.33 |

두 방법 모두 **2차 모형(Quadratic model)이 최선**임을 확인한다 — 훈련 데이터를 완벽하게 적합하지 않음에도 가장 낮은 테스트 오차를 달성한다 (반면 구간별 선형 모형은 정확하게 보간하지만 일반화가 가장 나쁘다).

### 6.6 과적합의 시각적 특징

오차 vs. 모형 복잡도를 도식화할 때 과적합의 특징적인 양상이 나타난다:

- **훈련 오차 (Training error)**: 모형 복잡도가 증가함에 따라 단조 **감소** (더 많은 모수는 항상 훈련 데이터를 더 잘 적합)
- **테스트 오차 (Test error)**: **U자형** — 처음에는 모형이 실제 패턴을 포착하면서 감소, 이후 모형이 노이즈를 적합하기 시작하면서 증가

훈련 오차와 테스트 오차 곡선이 **벌어지는** 지점이 과적합의 시작을 나타낸다. 최적의 모형 복잡도는 테스트 오차 곡선의 최솟값에 해당한다.

> **핵심 포인트:** 훈련 데이터를 완벽하게 적합하는 모형(구간별 보간 등)은 종종 새로운 데이터에서 성능이 저하된다. 목표는 잘 일반화되는 적절한 수준의 모형 복잡도를 찾는 것이다.

## 7. 검증 방법

### 7.1 방법 비교

| 방법 | 단점 | 장점 |
|------|------|------|
| **테스트 세트 (Test-set)** | 분산: 미래 성능에 대한 불안정한 추정 | 저비용 |
| **Leave-one-out (LOOCV)** | 고비용; 일부 이상한 동작 있음 | 데이터 낭비 없음 |
| **10-겹 (10-fold)** | 데이터의 10% 낭비; 테스트 세트보다 10배 비용 | 10%만 낭비; R배가 아닌 10배만 비용 증가 |
| **3-겹 (3-fold)** | 10-겹보다 더 낭비; 테스트 세트보다 더 비쌈 | 테스트 세트보다 약간 나음 |
| **R-겹 (R-fold)** | Leave-one-out과 동일 | - |

### 7.2 Leave-One-Out 교차 검증 (LOOCV)

알고리즘:
1. k = 1부터 R(데이터 포인트 수)까지:
   - (x_k, y_k)를 k번째 레코드로 함
   - 데이터셋에서 (x_k, y_k)를 임시로 제거
   - 나머지 R-1개 데이터 포인트로 훈련
   - (x_k, y_k)에 대한 오차를 기록
2. 평균 오차를 보고

$$CV_{(n)} = \frac{1}{n} \sum_{i=1}^{n} MSE_i$$

특성:
- 편향이 훨씬 적음; 테스트 오차율을 과대 추정하지 않는 경향
- 항상 동일한 결과를 산출 (분할의 무작위성 없음)
- 계산 집약적 (모형을 n번 적합해야 함)
- OLS에 대한 축약 공식: CV_(n) = (1/n) * sum((Y_i - Y_hat_i) / (1 - h_i))^2

### 7.3 K-겹 교차 검증 (K-Fold Cross-Validation)

- LOOCV가 계산 집약적이므로 사용
- 데이터 세트를 **K개의 겹 (fold)**으로 무작위 분할 (일반적으로 K = 5 또는 10)
- 각 겹이 번갈아 검증 세트 역할을 하고 나머지 K-1개 겹이 훈련 데이터로 사용
- 매번 보류된 겹에서 MSE를 계산
- K개의 추정치를 평균하여 최종 검증 오차율을 얻음

$$CV_{(K)} = \sum_{k=1}^{K} \frac{n_k}{n} MSE_k$$

### 7.4 편향-분산 트레이드오프 (Bias-Variance Trade-off)

- LOOCV는 K-겹 CV보다 계산 집약적
- LOOCV는 (K < n일 때) **편향이 더 적음** (편향 감소 관점에서 선호)
- LOOCV는 (K < n일 때) K-겹 CV보다 **분산이 더 높음**
- **K = 5 또는 K = 10**인 K-겹 CV를 사용하는 경향이 있으며, 이 값들이 편향과 분산의 균형을 맞추는 테스트 오차율 추정을 산출

> **핵심 포인트:** LOOCV는 K = n인 K-겹 교차 검증의 특수한 경우이다. 실제로는 K = 5 또는 K = 10이 좋은 편향-분산 트레이드오프를 제공하며 계산적으로 실현 가능하다.

## 8. 모델 성능 측정

### 8.1 가설 검정 프레임워크

혼동 행렬(Confusion matrix)은 가설 검정에서 유래한다. 통계적 검정 결과는 귀무가설을 기각하거나 기각하지 못한다:

- **H_0 (신호 부재)**: 귀무가설 — 효과 없음, 신호 없음
- **H_1 (신호 존재)**: 대립가설 — 효과 또는 신호가 존재

|  | H_1: 신호 존재 (Signal Present) | H_0: 신호 부재 (Signal Absent) |
|--|-------------------------------|-------------------------------|
| **탐지 (Detection)** | True Positive (TP) | False Positive (FP) — 제1종 오류 (Type I error) |
| **귀무 결과 (Null Result)** | False Negative (FN) — 제2종 오류 (Type II error) | True Negative (TN) |

주요 관계:
- **TPF (True Positive Fraction) = Sensitivity (민감도) = Power (검정력)** = TP / (TP + FN) — 참 신호를 올바르게 탐지하는 확률
- **FPF (False Positive Fraction) = 1 - Specificity (특이도)** = FP / (FP + TN) — 신호가 없을 때 잘못 탐지하는 확률

### 8.2 주요 측정 지표

| 측정 지표 | 공식 | 다른 명칭 |
|-----------|------|----------|
| **TPR** (True Positive Rate, 참 양성률) | TP / (TP + FN) | Sensitivity (민감도), Power (검정력), Recall (재현율), Hit Rate |
| **TNR** (True Negative Rate, 참 음성률) | TN / (TN + FP) | Specificity (특이도) |
| **PPV** (Positive Predictive Value, 양성 예측도) | TP / (TP + FP) | Precision (정밀도) |
| **NPV** (Negative Predictive Value, 음성 예측도) | TN / (TN + FN) | - |

### 8.3 오류 측정 지표

| 측정 지표 | 공식 | 관계 |
|-----------|------|------|
| **FNR** (False Negative Rate, 위음성률) | FN / (FN + TP) | = 1 - TPR (Miss Rate) |
| **FPR** (False Positive Rate, 위양성률) | FP / (FP + TN) | = 1 - TNR (Fall-out) |
| **FDR** (False Discovery Rate, 허위 발견률) | FP / (FP + TP) | = 1 - PPV |
| **FOR** (False Omission Rate, 허위 누락률) | FN / (FN + TN) | = 1 - NPV |

### 8.4 종합 성능

| 측정 지표 | 공식 |
|-----------|------|
| **ACC** (Accuracy, 정확도) | (TP + TN) / (TP + TN + FP + FN) |
| **ERR** (Error Rate, 오류율) | (FP + FN) / (TP + TN + FP + FN) = 1 - ACC |

> **핵심 포인트:** Sensitivity (민감도, TPR)와 Specificity (특이도, TNR)는 통계학과 데이터 과학에서 가장 널리 사용되는 측정 지표이다. 혼동 행렬을 이해하는 것은 모든 분류 모형을 평가하는 데 필수적이다.

## 9. K-최근접 이웃 (KNN)

### 9.1 개요

- 간단한 휴리스틱 분류 알고리즘
- **K개의 최근접 이웃 중 다수결 클래스**를 기반으로 새 데이터 포인트를 분류
- 큰 장점: **다중 클래스**를 고려할 수 있음
- K의 선택이 모형 복잡도에 영향:
  - K가 작을수록: 더 유연, 과적합(Overfitting) 위험
  - K가 클수록: 더 부드러운 결정 경계, 과소적합(Underfitting) 위험

### 9.2 거리 측정 (Distance Metric)

KNN은 "최근접"을 거리 측정으로 결정하며, 일반적으로 **유클리드 거리 (Euclidean distance)**를 사용한다:

$$d(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}$$

알고리즘은 새 포인트에서 모든 훈련 포인트까지의 거리를 계산한 다음, 가장 가까운 K개를 선택한다.

### 9.3 시각적 직관

녹색 원(?)을 두 개의 클래스(파란색 사각형, 빨간색 삼각형)로 분류하는 경우를 생각해 보자.
- **K = 3** (내부 원): 3개의 최근접 이웃이 빨간색 삼각형 2개 + 녹색 원 1개일 수 있어 "삼각형"으로 분류
- **K = 7** (외부 원): 7개의 최근접 이웃이 파란색 사각형 4개 + 빨간색 삼각형 3개를 포함할 수 있어 "사각형"으로 분류가 뒤바뀜

K의 선택은 결정 경계를 근본적으로 변화시킨다.

### 9.4 인스턴스 기반 (게으른) 학습

KNN은 **인스턴스 기반 (Instance-based)** 또는 **게으른 학습 (Lazy learning)**이라고 불리는데, 이는:
- 훈련 중 **명시적 모형을 구축하지 않고** — 단순히 **모든 훈련 데이터를 저장**
- 모든 계산이 분류 시점으로 지연됨
- **계산 비용**: 모든 새로운 쿼리에 대해 저장된 모든 훈련 포인트까지의 거리를 계산해야 하므로 예측 시 비용이 높음
- **메모리 비용**: 전체 훈련 데이터셋을 저장해야 함

> **핵심 포인트:** KNN은 기저 데이터 분포에 대한 가정을 하지 않는 인스턴스 기반(게으른) 학습 알고리즘이다. 단순히 훈련 데이터를 저장하고 최근접 이웃을 찾아 새 포인트를 분류한다.

## 요약

| 개념 | 설명 |
|------|------|
| 군집화 vs. 분류 (Clustering vs. Classification) | 군집화는 비지도(그룹 발견); 분류는 지도(사전 정의된 그룹 예측) |
| 분류 vs. 예측 (Classification vs. Prediction) | 본질적으로 동일; 분류는 구체적으로 범주형 결과를 지칭 |
| 2단계 프로세스 (Two-step process) | 1단계: 훈련 세트에서 모형 구축; 2단계: 예측을 위한 모형 사용 |
| 훈련 vs. 테스트 세트 (Training vs. Test set) | 훈련 세트는 모형을 구축; 테스트 세트는 평가; 반드시 독립적이어야 함 |
| 과적합 (Overfitting) | 모형이 훈련 데이터에 지나치게 적합하여 일반화에 실패; 과매개변수화가 원인 |
| 과소적합 (Underfitting) | 모형이 너무 단순; 훈련 오차와 테스트 오차 모두 큼 |
| LOOCV | 하나의 관측치를 빼고, 나머지로 훈련, n번 반복; 낮은 편향, 높은 분산 |
| K-겹 CV (K-Fold CV) | 데이터를 K개의 겹으로 분할; 편향-분산 균형을 위해 K = 5 또는 10 권장 |
| 혼동 행렬 (Confusion matrix) | TP, TN, FP, FN이 정확도, 민감도, 특이도, 정밀도의 기초가 됨 |
| KNN | K개의 최근접 이웃의 다수결로 분류; 다중 클래스 지원 |
