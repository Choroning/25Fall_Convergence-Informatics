## 1. 군집화 vs. 분류

### 1.1 군집화(Clustering)란?

군집 분석(Clustering Analysis)은 객체들(실험 대상 또는 그 특성)의 집합을 "군집(Cluster)"이라 불리는 부분집합으로 그룹화하는 과정이다. 이는 사전에 정의된 레이블 없이 원시 데이터로부터 학습하는 **비지도 학습(Unsupervised Learning)**의 가장 대표적인 형태이다.

- 객체 간의 **유사도(Similarity)**에 기반
- 군집 내 높은 유사도, 군집 간 낮은 유사도
- 구조 탐색 및 시각화, 이상치 식별, 결측값의 전역 패턴 탐지에 활용

### 1.2 군집화 vs. 분류

| 측면 | 군집화(Clustering) | 분류(Classification) |
|--------|-----------|----------------|
| 학습 유형 | 비지도(Unsupervised) | 지도(Supervised) |
| 클래스 | 사전에 알 수 없음 | 미리 정의됨 |
| 목표 | 클래스 발견 (새로운/미지의 클래스 식별) | 클래스 예측 (새 사례를 알려진 클래스에 배정) |
| 접근 방식 | 탐색적 분석 | 학습 데이터로부터 분류기 탐색 |

> **핵심 포인트:** 군집화에서는 클래스 정보가 없기 때문에 여러 유효한 답이 존재할 수 있다. 군집의 수는 분석 관점과 선택된 방법에 따라 달라진다.

## 2. 유사도와 거리

### 2.1 유사도 지수(Similarity Index)

서로 다른 객체들이 얼마나 **유사한지**를 나타내는 수치 값이다.

- 범위: `0 <= Similarity <= 1`
- 값이 높을수록 유사도가 큼

### 2.2 비유사도 지수(Dissimilarity Index)

객체들이 얼마나 **다른지**를 나타내는 수치 값이다.

- `Distance = 1 - Similarity = Dissimilarity`
- 객체 간 유사도가 높을수록 비유사도는 낮아짐

> **핵심 포인트:** 유사도 측정 방식은 사용되는 군집화 알고리즘보다 더 중요한 경우가 많다. 유사도/거리 척도의 선택이 군집화 결과를 근본적으로 결정한다.

## 3. 거리 척도

### 3.1 Manhattan Distance (맨해튼 거리, L1 Norm)

$$d_1(\mathbf{p}, \mathbf{q}) = \|\mathbf{p} - \mathbf{q}\|_1 = \sum_{i=1}^{n} |p_i - q_i|$$

**특성:**
- 이상치에 강건(Robust)
- 계산적으로 효율적
- 항상 최적해를 찾을 수 있는 것은 아님
- "시가지 거리"를 측정 (격자형 경로)

### 3.2 Euclidean Distance (유클리드 거리, L2 Norm)

$$d_{euc}(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}$$

**특성:**
- 수학적 최적화를 통해 최적해를 찾을 수 있음
- 이상치에 민감
- 두 점 사이의 직선 거리를 측정

### 3.3 L1 vs. L2 비교

| 특성 | L1 (Manhattan) | L2 (Euclidean) |
|----------|---------------|----------------|
| 이상치 강건성 | 강건 | 민감 |
| 계산 | 효율적 (계산적 방법) | 보다 수학적 |
| 최적해 | 항상 보장되지는 않음 | 최적해 탐색 가능 |
| 사용 사례 | 희소 데이터, 강건한 분석 | 범용 |

**L2는 전역 최적해를 찾을 수 있지만 L1은 찾을 수 없는 이유:**
- 두 척도 모두 부호(+/-)를 제거하는 방법을 사용한다. 거리는 음수가 될 수 없기 때문이다. 거리 범위: [0, 무한대).
- L2는 제곱을 사용하여 원점(0,0)을 포함한 모든 곳에서 매끄러운(미분 가능한) 표면을 생성한다. 이를 통해 경사 기반 최적화가 전역해를 찾을 수 있다.
- L1은 절댓값을 사용하는데, (0,0)에서 미분 불가능한 점을 생성한다. 도함수의 부호가 급격히 변하므로, 경사 기반 방법으로는 전역 최적해를 안정적으로 찾을 수 없다.

> **핵심 포인트:** L1 기반 방법은 일반적으로 L2 기반 방법보다 이상치에 더 강건하다. 단 하나의 이상치가 L2 기반 거리 행렬을 크게 왜곡할 수 있다.

## 4. 거리 행렬, 이상치 영향, 정규화

### 4.1 거리 행렬(Distance Matrix)

N x P 데이터 행렬(N개 표본, P개 특성)이 주어졌을 때:
- **표본 거리 행렬**: N x N (표본 간 거리)
- **특성 거리 행렬**: P x P (특성 간 거리)

**거리 행렬의 특성:**
- 항상 **대칭 행렬** (d(A,B) = d(B,A))
- 대각 원소는 항상 **0** (d(A,A) = 0)
- 거리 범위: **[0, 무한대)**

유클리드 거리 예제:

| | Ind1 | Ind2 | Ind3 | Ind4 |
|------|------|------|------|------|
| Ind1 | 0 | 2.828 | 3.162 | 5.099 |
| Ind2 | 2.828 | 0 | 1.414 | 3.162 |
| Ind3 | 3.162 | 1.414 | 0 | 2 |
| Ind4 | 5.099 | 3.162 | 2 | 0 |

### 4.2 이상치가 거리 행렬에 미치는 영향

단 하나의 이상치가 전체 거리 행렬을 깨뜨릴 수 있다. 이 현상은 L2 Norm에서 특히 두드러진다.

**예제:** 5명의 학생, 특성 (나이, 키, 성별). Student5의 키 = 300,000 (이상치, 본래 ~220이어야 함).

**이상치 없는 경우** (L2 표본 거리 행렬):

| | St1 | St2 | St3 | St4 | St5 |
|-----|------|------|------|------|------|
| St1 | 0 | | | | |
| St2 | 1.97 | 0 | | | |
| St3 | 5.76 | 7.17 | 0 | | |
| St4 | 12.37 | 13.85 | 6.68 | 0 | |
| St5 | 50.46 | 52.09 | 44.94 | 38.31 | 0 |

**이상치 있는 경우** (Student5 키 = 300,000, L2):

| | St1 | St2 | St3 | St4 | St5 |
|-----|------|------|------|------|--------|
| St1 | 0 | | | | |
| St2 | 1.97 | 0 | | | |
| St3 | 5.76 | 7.17 | 0 | | |
| St4 | 12.37 | 13.85 | 6.68 | 0 | |
| St5 | 299,829 | 299,831 | 299,824 | 299,817 | 0 |

- St5와 관련된 거리가 ~50에서 ~299,000으로 폭증
- L1도 영향을 받지만 덜 심각함 (거리가 ~299,818까지 증가)
- 이것은 **거리 행렬을 계산하기 전에 이상치 탐지가 중요한 이유**를 보여줌

### 4.3 정규화 (Z-scoring)

거리를 계산하기 전에, 비교 가능성을 확보하기 위해 특성을 정규화해야 하는 경우가 많다:

- **Z-score 정규화**: 각 특성별로 평균을 빼고 표준편차로 나눔
- 스케일이 큰 특성이 거리 계산을 지배하는 것을 방지

> **핵심 포인트:** 정규화 없이는 크기가 큰 특성이 거리 계산에 불균형적으로 큰 영향을 미치게 된다.

## 5. 상관관계 기반 거리

### 5.1 공분산(Covariance)

$$Cov(X, Y) = E[(X - E[X])(Y - E[Y])]$$

- 공분산은 두 변수의 평균으로부터의 편차의 기대 곱으로, 결합 변동성을 정량화함
- 공분산은 **스케일 의존적**: 어느 한 변수를 재조정하면 그 크기가 변함
- 공분산을 표준편차의 곱으로 나누면 스케일 불변인 **Pearson 상관계수**가 됨

### 5.2 Pearson 상관계수 (Pearson's Correlation)

$$r_{xy} = \frac{Cov(x, y)}{\sigma_x \cdot \sigma_y} = \frac{S_{xy}}{\sqrt{S_{xx} S_{yy}}}$$

- L2 Norm 기반 (수학적 방법)
- **선형(Linear)** 관계를 측정
- 범위: [-1, 1]
- 이상치에 민감
- |r| = 1은 완벽한 선형 관계에서만 성립

### 5.3 Spearman 상관계수 (Spearman's Correlation)

$$r_s = \rho_{\text{rg}_X, \text{rg}_Y} = \frac{\text{cov}(\text{rg}_X, \text{rg}_Y)}{\sigma_{\text{rg}_X} \cdot \sigma_{\text{rg}_Y}}$$

- 순위(Rank) 기반 (L1 개념, 계산적 방법)
- **단조(Monotonic)** 관계를 포착 (선형뿐 아니라, 예: y = x^3)
- 이상치에 강건 (순위가 극단값을 흡수하므로)

### 5.4 Pearson vs. Spearman: 구체적 예제

**예제 1 — 이상치 없는 데이터 (4명의 직원):**

| Employee | Hrs. Worked (Monthly) | Tasks Completed | Satisfaction |
|----------|----------------------|-----------------|--------------|
| A | 20 | 10 | 5.0 |
| B | 30 | 15 | 8.0 |
| C | 40 | 20 | 9.0 |
| D | 50 | 25 | 9.5 |

Pearson 상관계수:

| | HW | TC | S |
|-----|------|------|------|
| HW | 1.0 | | |
| TC | 1.0 | 1.0 | |
| S | 0.96 | 0.96 | 1.0 |

Spearman 상관계수:

| | HW | TC | S |
|-----|------|------|------|
| HW | 1.0 | | |
| TC | 1.0 | 1.0 | |
| S | 1.0 | 1.0 | 1.0 |

- HW-TC는 완벽한 선형 관계이므로, 두 방법 모두 1.0을 정확히 측정
- HW-S는 단조적이지만 완벽한 선형은 아님; Pearson은 0.96으로 과소평가하는 반면, Spearman은 완벽한 단조 추세를 1.0으로 정확히 포착

**예제 2 — 이상치 있는 데이터 (직원 D: Tasks = 5):**

| Employee | Hrs. Worked (Monthly) | Tasks Completed | Satisfaction |
|----------|----------------------|-----------------|--------------|
| A | 20 | 10 | 5.0 |
| B | 30 | 15 | 8.0 |
| C | 40 | 20 | 9.0 |
| D | 50 | **5** | 9.5 |

Pearson 상관계수:

| | HW | TC | S |
|-----|-------|-------|------|
| HW | 1.0 | | |
| TC | **-0.45** | 1.0 | |
| S | 0.96 | **-0.66** | 1.0 |

Spearman 상관계수:

| | HW | TC | S |
|-----|------|------|------|
| HW | 1.0 | | |
| TC | **0.4** | 1.0 | |
| S | 1.0 | **0.4** | 1.0 |

- 단 하나의 이상치로 인해 Pearson HW-TC가 **1.0에서 -0.45로** 떨어졌으며, 부호가 완전히 뒤바뀌어 근본적인 양의 추세를 잘못 표현
- Spearman HW-TC는 **1.0에서 0.4로**만 떨어졌으며, 여전히 양의 관계를 올바르게 식별
- 이는 Pearson의 극단적인 이상치 민감도 대비 Spearman의 강건성을 보여줌

### 5.5 상관관계 기반 거리 척도(Correlation-Based Distance Metric)

$$D(X, Y) = 1 - |cor(X, Y)|$$

- 범위: [0, 1]

**이 변환이 필요한 이유:**
- 상관계수의 범위는 [-1, 1]이지만, 거리의 범위는 [0, 무한대)여야 함
- 상관계수는 **음수**일 수 있으므로 직접 거리로 사용할 수 없음
- 변환 D = 1 - |cor|은 [0, 1]로 매핑되며, 거리 요건을 충족
- **한계:** 방향 정보가 손실됨 — 원래 상관계수가 양수인지 음수인지 판단할 수 없음

**상관관계 기반 거리의 장점:**
- 상관계수는 **스케일 불변**이므로 정규화가 필요 없음
- 데이터 형태를 왜곡할 수 있는 정규화가 필요한 L1/L2 거리와 달리, 상관관계는 원본 데이터에 직접 적용 가능

### 5.6 Norm 기반 vs. 상관관계 기반 거리

| 특성 | Norm 기반 (L1/L2) | 상관관계 기반 |
|----------|-------------------|-------------------|
| 범위 | [0, 무한대) | [0, 1] (변환 후) |
| 측정 대상 | 크기 차이 | 패턴 유사성 |
| 스케일 민감도 | 스케일 및 이상치에 매우 민감 | 스케일 불변 |
| 정규화 | 대개 필요 (데이터 왜곡 가능) | 불필요 |
| 방향 정보 | 방향 개념 없음 | 절댓값 사용 시 손실 |
| 사용 대상 | 절대값 차이 | 데이터의 추세/형태 |
| 한계 | 높은 스케일 민감도 | 비단조 관계를 놓칠 수 있음 |

> **핵심 포인트:** Pearson 상관계수는 선형 관계를 포착하며 이상치에 민감한 반면, Spearman 상관계수는 단조 관계를 포착하며 이상치에 강건하다.

## 7. Hierarchical Clustering (계층적 군집화)

### 7.1 개요

HC는 여러 수준의 세분도(Granularity)에서 데이터를 그룹화하는 **덴드로그램(Dendrogram)**이라 불리는 이진 트리 구조를 생성한다.

- **병합적 (Agglomerative, 상향식)**: 각 객체를 자체 군집으로 시작하여, 가장 가까운 쌍을 병합
- **분할적 (Divisive, 하향식)**: 모든 객체를 하나의 군집으로 시작하여, 재귀적으로 분할
- 병합적 방법이 가장 일반적으로 사용됨

### 7.2 HCA 4단계

계층적 군집 분석(Hierarchical Clustering Analysis)을 수행하는 과정은 네 가지 주요 단계로 나눌 수 있다:

1. **거리 행렬 생성**
2. **덴드로그램 생성** (연결법을 사용)
3. **덴드로그램에서 그룹 정보 추출** (선택한 높이에서 자르기)
4. **군집화 결과 평가**

### 7.3 병합적 HC 알고리즘

1. q개의 객체가 q개의 군집을 형성하도록 함 (각 객체가 자체 군집)
2. 가장 작은 거리를 가진 두 군집을 병합 (#군집 = q - 1)
3. 모든 군집이 계층적 트리(덴드로그램)로 연결될 때까지 반복

### 7.4 덴드로그램 특성

- 최종 군집이 **루트(Root)**이고 각 데이터 항목이 **리프(Leaf)**
- 막대의 **높이**는 항목들이 얼마나 가까운지를 나타냄
- 원하는 높이에서 덴드로그램을 잘라 군집의 수를 결정

### 7.5 연결법(Linkage Methods)

연결법은 군집 G와 H 사이의 거리를 측정하는 방법을 결정한다:

| 연결법 | 공식 | 설명 |
|---------|---------|-------------|
| **Single** | d_S(G,H) = min d(z_i, z_j) | 최소 거리 (최근접 이웃) |
| **Complete** | d_C(G,H) = max d(z_i, z_j) | 최대 거리 (최원접 이웃) |
| **Average** | d_A(G,H) = (1/n_G*n_H) * sum d(z_i, z_j) | 모든 쌍별 거리의 평균 |

### 7.6 HC 풀이 예제: 6개 포인트 (A-F), 3가지 연결법 모두

**주어진 조건:** 6개의 2D 데이터 포인트 (A-F). 초기 **L1 (Manhattan) 거리 행렬**:

| L1 | A | B | C | D | E | F |
|----|-----|-----|-----|-----|-----|-----|
| A | 0 | | | | | |
| B | 5 | 0 | | | | |
| C | 5 | 4 | 0 | | | |
| D | 15 | 10 | 10 | 0 | | |
| E | 16 | 11 | 11 | 1 | 0 | |
| F | 19 | 14 | 14 | 4 | 3 | 0 |

가장 작은 거리는 d(D,E) = 1이므로, 세 가지 방법 모두 {D, E}를 병합하는 것으로 시작한다.

---

#### 7.6.1 Single Linkage (최근접 이웃법)

**갱신 규칙:** d(병합, X) = min[d(각 구성원, X)]

**단계 1:** {D, E} 병합 (거리 = 1). min을 사용하여 거리 갱신:
- d(DE, A) = min[d(D,A), d(E,A)] = min[15, 16] = 15
- d(DE, B) = min[d(D,B), d(E,B)] = min[10, 11] = 10
- d(DE, C) = min[d(D,C), d(E,C)] = min[10, 11] = 10
- d(DE, F) = min[d(D,F), d(E,F)] = min[4, 3] = 3

| Step 1 | A | B | C | DE | F |
|--------|-----|-----|-----|------|-----|
| A | 0 | | | | |
| B | 5 | 0 | | | |
| C | 5 | 4 | 0 | | |
| DE | 15 | 10 | 10 | 0 | |
| F | 19 | 14 | 14 | **3** | 0 |

**단계 2:** {F, DE} 병합 (거리 = 3). min을 사용하여 거리 갱신:
- d(DEF, A) = min[d(D,A), d(E,A), d(F,A)] = min[15, 16, 19] = 15
- d(DEF, B) = min[d(D,B), d(E,B), d(F,B)] = min[10, 11, 14] = 10
- d(DEF, C) = min[d(D,C), d(E,C), d(F,C)] = min[10, 11, 14] = 10

| Step 2 | A | B | C | DEF |
|--------|-----|-----|-----|------|
| A | 0 | | | |
| B | 5 | 0 | | |
| C | 5 | **4** | 0 | |
| DEF | 15 | 10 | 10 | 0 |

**단계 3:** {B, C} 병합 (거리 = 4).
- d(BC, A) = min[d(B,A), d(C,A)] = min[5, 5] = 5
- d(BC, DEF) = min[d(B,DEF), d(C,DEF)] = min[10, 10] = 10

| Step 3 | A | BC | DEF |
|--------|-----|------|------|
| A | 0 | | |
| BC | **5** | 0 | |
| DEF | 15 | 10 | 0 |

**단계 4:** {A, BC} 병합 (거리 = 5).
- d(ABC, DEF) = min[d(A,DEF), d(BC,DEF)] = min[15, 10] = 10

| Step 4 | ABC | DEF |
|--------|------|------|
| ABC | 0 | |
| DEF | **10** | 0 |

**Single Linkage 병합 순서:** {D,E} at 1 --> {F,DE} at 3 --> {B,C} at 4 --> {A,BC} at 5 --> {ABC,DEF} at 10

---

#### 7.6.2 Complete Linkage (최원접 이웃법)

**갱신 규칙:** d(병합, X) = max[d(각 구성원, X)]

**단계 1:** {D, E} 병합 (거리 = 1). max를 사용하여 거리 갱신:
- d(DE, A) = max[d(D,A), d(E,A)] = max[15, 16] = 16
- d(DE, B) = max[d(D,B), d(E,B)] = max[10, 11] = 11
- d(DE, C) = max[d(D,C), d(E,C)] = max[10, 11] = 11
- d(DE, F) = max[d(D,F), d(E,F)] = max[4, 3] = 4

| Step 1 | A | B | C | DE | F |
|--------|-----|-----|-----|------|-----|
| A | 0 | | | | |
| B | 5 | 0 | | | |
| C | 5 | **4** | 0 | | |
| DE | 16 | 11 | 11 | 0 | |
| F | 19 | 14 | 14 | 4 | 0 |

**단계 2:** {B, C} 병합 (거리 = 4, 최소 거리 중 하나). max를 사용하여 거리 갱신:
- d(BC, A) = max[d(B,A), d(C,A)] = max[5, 5] = 5
- d(BC, DE) = max[d(B,DE), d(C,DE)] = max[11, 11] = 11
- d(BC, F) = max[d(B,F), d(C,F)] = max[14, 14] = 14

| Step 2 | A | BC | DE | F |
|--------|-----|------|------|-----|
| A | 0 | | | |
| BC | **5** | 0 | | |
| DE | 16 | 11 | 0 | |
| F | 19 | 14 | 4 | 0 |

**단계 3:** {DE, F} 병합 (거리 = 4).
- d(DEF, A) = max[d(DE,A), d(F,A)] = max[16, 19] = 19
- d(DEF, BC) = max[d(DE,BC), d(F,BC)] = max[11, 14] = 14

| Step 3 | A | BC | DEF |
|--------|-----|------|------|
| A | 0 | | |
| BC | 5 | 0 | |
| DEF | 19 | **14** | 0 |

**단계 4:** {A, BC} 병합 (거리 = 5).
- d(ABC, DEF) = max[d(A,DEF), d(BC,DEF)] = max[19, 14] = 19

| Step 4 | ABC | DEF |
|--------|------|------|
| ABC | 0 | |
| DEF | **19** | 0 |

**Complete Linkage 병합 순서:** {D,E} at 1 --> {B,C} at 4 --> {DE,F} at 4 --> {A,BC} at 5 --> {ABC,DEF} at 19

---

#### 7.6.3 Average Linkage (평균 연결법)

**갱신 규칙:** d(병합, X) = avg[d(각 구성원, X)] (모든 쌍별 거리의 평균)

**단계 1:** {D, E} 병합 (거리 = 1). 평균을 사용하여 거리 갱신:
- d(DE, A) = avg[d(D,A), d(E,A)] = avg[15, 16] = 15.5
- d(DE, B) = avg[d(D,B), d(E,B)] = avg[10, 11] = 10.5
- d(DE, C) = avg[d(D,C), d(E,C)] = avg[10, 11] = 10.5
- d(DE, F) = avg[d(D,F), d(E,F)] = avg[4, 3] = 3.5

| Step 1 | A | B | C | DE | F |
|--------|------|-----|-----|------|-----|
| A | 0 | | | | |
| B | 5 | 0 | | | |
| C | 5 | 4 | 0 | | |
| DE | 15.5 | 10.5 | 10.5 | 0 | |
| F | 19 | 14 | 14 | **3.5** | 0 |

**단계 2:** {F, DE} 병합 (거리 = 3.5). 평균을 사용하여 거리 갱신 (현재 3개 구성원: D, E, F):
- d(DEF, A) = avg[d(D,A), d(E,A), d(F,A)] = avg[15, 16, 19] = 16.67
- d(DEF, B) = avg[d(D,B), d(E,B), d(F,B)] = avg[10, 11, 14] = 11.67
- d(DEF, C) = avg[d(D,C), d(E,C), d(F,C)] = avg[10, 11, 14] = 11.67

| Step 2 | A | B | C | DEF |
|--------|------|-----|-----|-------|
| A | 0 | | | |
| B | 5 | 0 | | |
| C | 5 | **4** | 0 | |
| DEF | 16.67 | 11.67 | 11.67 | 0 |

**단계 3:** {B, C} 병합 (거리 = 4).
- d(BC, A) = avg[d(B,A), d(C,A)] = avg[5, 5] = 5
- d(BC, DEF) = avg[d(B,DEF), d(C,DEF)] = avg[11.67, 11.67] = 11.67

| Step 3 | A | BC | DEF |
|--------|-----|------|-------|
| A | 0 | | |
| BC | **5** | 0 | |
| DEF | 16.67 | 11.67 | 0 |

**단계 4:** {A, BC} 병합 (거리 = 5).
- d(ABC, DEF) = 모든 3x3 쌍별 거리의 평균 = 13.33

| Step 4 | ABC | DEF |
|--------|------|-------|
| ABC | 0 | |
| DEF | **13.33** | 0 |

**Average Linkage 병합 순서:** {D,E} at 1 --> {F,DE} at 3.5 --> {B,C} at 4 --> {A,BC} at 5 --> {ABC,DEF} at 13.33

---

#### 7.6.4 덴드로그램 자르기 예제

**Average Linkage** 덴드로그램을 사용하여, **높이 = 8**에서 자르면 두 개의 군집이 생성된다:
- **군집 1:** {F, D, E} (높이 1과 3.5에서 병합, 모두 8 미만)
- **군집 2:** {A, B, C} (높이 4와 5에서 병합, 모두 8 미만)

군집 간 병합은 13.33에서 이루어지며 이는 절단선 위에 있으므로, 두 그룹은 분리된 상태로 유지된다.

어디에서 자를지에 대한 단 하나의 정답은 없다; 선택은 분석 목표에 따라 달라지지만, 평가 지표가 결정을 안내하는 데 도움이 될 수 있다.

### 7.7 군집 레이블(Cluster Labels)

군집 레이블은 임의의 이름을 가진 **명목형 범주 변수(Nominal Categorical Variables)**이다. 레이블 자체는 고유한 의미를 갖지 않으며, 그룹화만이 중요하다. (예: [A, A, B]는 [B, B, A]와 동일)

### 7.8 장단점

| 장점 | 단점 |
|------|------|
| 모든 수준의 세분도 탐색 가능 (1~N개 군집) | O(N^2) 계산 비용 |
| 군집 수를 사전 지정할 필요 없음 | 저장 요구량: (N^2 - N) / 2 |
| 해석 가능한 덴드로그램 생성 | 노이즈와 이상치에 민감 |

> **핵심 포인트:** HC는 여러 세분도 수준에서 데이터 구조를 탐색하는 데 이상적이지만, O(N^2) 비용과 (N^2 - N) / 2 크기의 거리 행렬 저장 필요성으로 인해 대규모 데이터셋에서는 계산 비용이 높아진다.

## 8. K-Means Clustering (K-평균 군집화)

### 8.1 개요

K-means는 군집 내 분산을 최소화하여 데이터를 K개의 그룹으로 나누는 **하향식 분할(Top-down Partitioning)** 방법이다. **EM (Expectation-Maximization) 알고리즘**의 대표적인 예이다.

### 8.2 알고리즘

1. 군집의 수 K를 **선택**
2. 군집 중심(Centroid)을 무작위로 **초기화**
3. 각 데이터 포인트를 가장 가까운 군집 중심에 **배정**
4. 중심점을 각 군집에 속한 모든 점의 평균으로 **갱신**
5. 수렴할 때까지 (새로운 재배정이 없을 때까지) 3-4단계를 **반복**

### 8.3 수학적 정의

m_k(t) = 단계 t에서 k번째 군집의 중심점, C_i(t) = 단계 t에서 i번째 객체의 군집 배정이라 하자.

**배정 단계:**

$$C_i(t) = \underset{1 \leq k \leq K}{\text{argmin}} \; d(z_i, m_k(t))$$

**수렴 기준** (총 군집 내 분산):

$$V(t) = \sum_{k=1}^{K} \sum_{i: C_i(t)=k} (z_i - m_k(t))^2$$

### 8.4 문제점 및 한계

- **무작위 초기화**: 다른 실행이 다른 군집을 생성할 수 있음
- **경성 배정(Hard Assignment)**: 각 데이터 포인트는 정확히 하나의 군집에만 소속
- **형태 가정**: 암묵적으로 대략 구형의 군집을 가정
- **K를 지정해야 함**: 군집의 수를 사전에 선택해야 함
- **지역 최적해(Local Optimum)**: 전역 최적해를 찾지 못함 (지역 최솟값에 수렴)
- **군집 내 유사도 정보 없음**: HC와 달리, K-means는 군집의 내부 구조에 대한 정보를 제공하지 않음

> **핵심 포인트:** K-means는 대규모 데이터셋에 효율적이지만, K를 사전 지정해야 하며 초기화에 따라 다른 솔루션으로 수렴할 수 있다. 이를 완화하기 위해 여러 번의 무작위 시작을 사용해야 한다.
